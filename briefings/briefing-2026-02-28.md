## 주요 신규 기능 & 실전 활용

### 1. HTTP Hooks — 셸 명령 대신 URL로 JSON 전송 (v2.1.63)

HTTP 훅이 새로 추가되었습니다. 기존 훅은 로컬 셸 명령만 실행할 수 있었지만, 이제 지정한 URL로 JSON을 POST하고 응답 JSON을 받아 처리할 수 있습니다. Slack·Discord 알림, 외부 CI/CD 트리거, 커스텀 감사 로그 서버 연동 등 셸 스크립트 없이도 외부 시스템과 연결할 수 있습니다.

```json
// .claude/settings.json 예시
{
  "hooks": {
    "PostToolUse": {
      "url": "https://your-server.com/webhook",
      "method": "POST"
    }
  }
}
```

[GitHub Releases](https://github.com/anthropics/claude-code/releases)

### 2. `/simplify` & `/batch` — 번들 슬래시 명령어 (v2.1.63)

두 가지 새로운 번들 슬래시 명령어가 추가되었습니다. `/simplify`는 복잡한 코드를 간결하게 리팩토링하는 데 사용되며, `/batch`는 여러 파일에 대한 반복 작업을 일괄 처리할 때 활용됩니다. 기존에 긴 프롬프트로 설명해야 했던 작업을 명령어 하나로 실행할 수 있습니다.

```bash
/simplify  # 현재 컨텍스트의 코드를 간결하게 리팩토링
/batch     # 여러 파일에 대한 반복 작업 일괄 실행
```

[GitHub Releases](https://github.com/anthropics/claude-code/releases)

### 3. Git Worktree 간 프로젝트 설정·자동 메모리 공유 (v2.1.63)

동일 레포지토리의 git worktree들이 프로젝트 설정과 자동 메모리를 공유합니다. 이전에는 worktree마다 설정이 격리되어 있어 동일한 CLAUDE.md, 자동 메모리, 프로젝트 설정을 각각 관리해야 했지만, 이제 한 worktree에서 학습한 맥락이 같은 레포지토리의 모든 worktree에 자동으로 적용됩니다. 멀티 브랜치 병렬 작업 워크플로우에서 특히 유용합니다.

[GitHub Releases](https://github.com/anthropics/claude-code/releases)

## 개발자 워크플로우 팁

### v2.1.62 Stale Context 회귀 — 긴 세션 사용자 주의

v2.1.62(2/27 배포)에서 서버 사이드 KV 캐시 개선이 적용되었으나, 컨텍스트 압축(compaction) 이벤트 시 캐시 무효화가 누락되어 P1 회귀가 발생했습니다. 컨텍스트 압축 후 모델이 이전 상태를 기반으로 응답하며, 파일이 존재하는데 "없다"고 주장하거나, 사용자의 지시를 무시하고 이전 작업 계획을 계속 실행하는 증상이 보고되었습니다.

**대응 방법:**
1. **즉시 v2.1.63으로 업데이트** — 오늘 릴리스된 버전에 관련 수정이 포함되어 있습니다
2. **긴 세션은 주기적으로 `/clear`로 초기화** — 컨텍스트 압축이 발생하기 전에 세션을 리셋
3. **모델이 비정상 응답을 보이면 새 세션 시작** — stale 캐시는 새 세션에서 발생하지 않습니다

[GitHub Issue #29230](https://github.com/anthropics/claude-code/issues/29230)

### v2.1.63 메모리 누수 대규모 수정 — 장시간 세션 안정성 대폭 개선

v2.1.63에서 8건 이상의 메모리 누수가 한꺼번에 수정되었습니다. 장시간 Claude Code를 실행하는 사용자에게 직접적인 영향이 있습니다.

**수정된 누수 목록:**
- Git 루트 감지 캐시 무한 증가
- JSON 파싱 캐시 무한 증가
- Bash 명령 접두사 캐시
- MCP 서버 fetch 캐시 (재연결 시 미정리)
- 팀메이트 메시지 AppState 미해제 (압축 후에도 유지)
- Bridge 폴링 루프 리스너 누수
- MCP OAuth 플로우 리스너 누수
- 인터랙티브 권한 핸들러 리스너 누수
- 훅 설정 메뉴 탐색 시 메모리 누수
- IDE 호스트 IP 감지 캐시 포트 간 잘못된 공유
- WebSocket 전송 재연결 리스너 누수

장시간 세션(특히 서브에이전트·팀메이트 사용 시)에서 메모리 사용량이 지속적으로 증가하던 문제가 해소됩니다.

## 생태계 & 플러그인

### claude.ai MCP 서버 비활성화 옵션 추가 (v2.1.63)

`ENABLE_CLAUDEAI_MCP_SERVERS=false` 환경변수가 추가되어, claude.ai에서 제공하는 MCP 서버를 Claude Code에서 사용하지 않도록 선택할 수 있습니다. 기업 환경에서 승인된 MCP 서버만 사용하도록 제한하거나, 불필요한 도구 로딩을 줄여 컨텍스트 효율을 높이고 싶을 때 유용합니다.

```bash
# claude.ai MCP 서버 비활성화
export ENABLE_CLAUDEAI_MCP_SERVERS=false
```

### MCP OAuth 수동 URL 붙여넣기 폴백 (v2.1.63)

MCP 서버 OAuth 인증 시 자동 localhost 리다이렉트가 작동하지 않을 때 콜백 URL을 수동으로 붙여넣어 인증을 완료할 수 있는 폴백이 추가되었습니다. SSH 원격 환경이나 포트 포워딩이 제한된 환경에서 MCP OAuth 인증이 실패하던 문제를 해결합니다.

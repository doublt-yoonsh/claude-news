## 주요 신규 기능 & 실전 활용

### 1. 자동 메모리 — Claude가 스스로 맥락을 기억 (v2.1.59)

Claude Code가 대화 중 유용한 맥락을 자동으로 메모리에 저장합니다. 프로젝트 규칙, 빌드 명령, 자주 쓰는 패턴 등을 반복 설명할 필요가 줄어듭니다. 기존에는 CLAUDE.md에 수동 기록하거나 `/memory` 명령으로 직접 관리해야 했지만, 이제 작업 과정에서 자연스럽게 학습됩니다.

```bash
# 메모리 확인 및 관리
/memory
```

[GitHub Releases](https://github.com/anthropics/claude-code/releases)

### 2. `/copy` — 코드블록 인터랙티브 선택기 (v2.1.59)

응답에 코드 블록이 포함되어 있을 때 `/copy` 명령으로 개별 코드 블록 또는 전체 응답을 선택하여 클립보드에 복사할 수 있습니다. 여러 코드 블록이 섞인 긴 응답에서 원하는 부분만 정확히 추출할 때 유용합니다.

```bash
# 코드블록이 포함된 응답 후
/copy    # 인터랙티브 선택기 표시
```

[GitHub Releases](https://github.com/anthropics/claude-code/releases)

### 3. Compound Bash "Always Allow" 개선 (v2.1.59)

`cd /tmp && git fetch && git push` 같은 복합 명령어의 "always allow" 접두사 제안이 개선되었습니다. 이전에는 전체 명령어를 하나의 단위로 취급했지만, 이제 서브 명령별로 더 세밀한 접두사를 계산합니다. 반복적인 복합 명령 승인 피로가 줄어듭니다.

[GitHub Releases](https://github.com/anthropics/claude-code/releases)

## 개발자 워크플로우 팁

### Pencil.dev + Claude Code — 디자인에서 프로덕션 코드까지 20분

Pencil.dev는 MCP를 통해 Claude Code와 네이티브로 연결되는 무료 디자인 캔버스입니다. Figma에서 디자인을 가져오면 벡터, 텍스트, 스타일이 그대로 보존되며, Claude가 디자인 컨텍스트를 직접 읽어 HTML/CSS/React 컴포넌트를 생성합니다. 핵심은 `.pen` 파일이 Git 리포지토리에 저장되어 디자인과 코드가 같은 버전 관리 하에 놓인다는 점입니다. 디자인 변경을 커밋하면 코드 롤백 시 디자인도 함께 롤백되어 드리프트가 원천 차단됩니다.

초기 디자인 컨셉 → 배포 가능한 코드까지 기존 대비 50~60% 빠르다는 보고가 있습니다. 현재 얼리 액세스 무료 사용 가능합니다.

[Pencil.dev + Claude Code Workflow](https://atalupadhyay.wordpress.com/2026/02/25/pencil-dev-claude-code-workflow-from-design-to-production-code-in-minutes/) | [Pencil.dev 문서](https://docs.pencil.dev/getting-started/ai-integration)

### Cloudflare, Claude로 Next.js를 1주일 만에 Vite로 재구현

Cloudflare 엔지니어가 Claude Code를 활용하여 Next.js 기반 인프라를 Vite로 완전히 재구현한 `vinext` 프로젝트를 공개했습니다. 1주일이라는 짧은 기간에 프레임워크 마이그레이션이 가능했던 핵심은 Claude Code의 대규모 코드베이스 맥락 이해와 에이전틱 리팩토링 능력이었다고 밝혔습니다. 대규모 코드 마이그레이션 작업에 Claude Code를 활용하는 실전 사례로 주목받고 있습니다.

## 보안/제한 이슈

### Anthropic, 핵심 안전 서약 폐기 — RSP 3.0 발효 (2/25)

Anthropic이 2023년부터 유지해온 핵심 안전 서약을 폐기했습니다. 기존 정책은 "안전 조치가 충분히 검증되지 않으면 더 강력한 모델을 절대 훈련하지 않겠다"는 엄격한 약속이었지만, RSP 3.0(2/24 발효)에서는 이를 구속력 없는 "공개 목표"로 대체했습니다.

변경 배경으로 세 가지를 꼽았습니다: (1) 위험 임계값의 모호한 경계, (2) 반규제 정치 분위기, (3) 업계 공동 대응 없이는 달성 불가능한 상위 단계 요건. 새 프레임워크는 Anthropic이 독자적으로 추진할 안전 조치와 업계 전체가 채택해야 할 조치를 이원화하며, 3~6개월 주기로 Frontier Safety Roadmap과 Risk Report를 공개합니다.

[TIME 독점 보도](https://time.com/7380854/exclusive-anthropic-drops-flagship-safety-pledge/) | [Anthropic RSP v3.0](https://anthropic.com/responsible-scaling-policy/rsp-v3-0) | [Bloomberg](https://www.bloomberg.com/news/articles/2026-02-25/anthropic-adds-caveat-to-ai-safety-policy-in-race-against-rivals)

### 미 국방부, Anthropic에 AI 가드레일 해제 최후통첩 (2/24~25)

피트 헤그세스 국방장관이 Anthropic에 군사용 AI 제한 해제를 요구하며 2억 달러 규모 계약 취소와 국방생산법(Defense Production Act) 발동을 위협했습니다. Anthropic은 자율 무기의 최종 타격 결정과 미국 시민 대상 대규모 감시라는 두 가지 '레드 라인'을 유지하고 있지만, 국방부는 "공급망 리스크" 지정(통상 적국에 적용)까지 언급하며 압박 중입니다. 금요일 오후 5시 마감 시한이 설정되었습니다.

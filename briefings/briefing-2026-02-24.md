# Claude Code 데일리 브리핑 - 2026-02-24

## 💡 사용 팁

이번 주는 에이전트 격리와 작업 환경 통합에 관한 업데이트가 눈에 띄었어요.

- **--worktree 플래그로 안전한 병렬 작업** (2/20)
  v2.1.49에서 `--worktree`(`-w`) 플래그가 추가되어, 에이전트를 격리된 git worktree에서 실행할 수 있게 됐어요. 여러 에이전트가 동시에 다른 브랜치에서 작업해도 충돌 걱정이 없으니, 대규모 리팩터링이나 병렬 기능 개발에 바로 적용해볼 만합니다.
  → [Claude Code Releases](https://github.com/anthropics/claude-code/releases)

- **claude.ai MCP 커넥터를 CLI에서도 사용 가능** (2/19)
  v2.1.46부터 claude.ai 웹에서 설정한 MCP 커넥터를 Claude Code CLI에서도 바로 쓸 수 있어요. 웹과 터미널을 오가며 동일한 도구 세트를 활용할 수 있다는 점이 반가운 변화입니다.
  → [Releasebot - Claude Code](https://releasebot.io/updates/anthropic/claude-code)

- **백그라운드 에이전트 제어 강화** (2/20)
  에이전트 정의에 `background: true`를 설정하면 항상 백그라운드로 실행되고, Ctrl+F(두 번 눌러 확인)로 모든 백그라운드 에이전트를 한 번에 종료할 수 있게 됐어요. ESC를 눌러도 백그라운드 에이전트는 계속 돌아가므로, 메인 작업과 분리해서 활용하기 좋은 구조예요.
  → [Claude Code Releases](https://github.com/anthropics/claude-code/releases)

## 🔒 보안/제한 이슈

- **Claude Desktop Extension(DXT) 제로클릭 RCE 취약점, Anthropic 수정 거부** (2/9 공개, 보도 지속)
  50개 이상의 Claude DXT에서 CVSS 10.0 등급의 원격 코드 실행 취약점이 발견됐어요. 구글 캘린더 이벤트 하나로 시스템을 장악할 수 있는 심각한 문제인데, DXT가 샌드박스 없이 전체 시스템 권한으로 실행되는 것이 근본 원인이에요. Anthropic은 "현재 위협 모델 범위 밖"이라며 수정을 거부한 상태라 논란이 이어지고 있습니다.
  → [Infosecurity Magazine](https://www.infosecurity-magazine.com/news/zeroclick-flaw-claude-dxt/) | [The Register](https://www.theregister.com/2026/02/11/claude_desktop_extensions_prompt_injection/)

## 主要な新機能と実践活用

### Focus Viewトグル — Ctrl+O（v2.1.97）

v2.1.97でNO_FLICKERモードに**Focus View**が追加されました。`Ctrl+O`でトグルすると、プロンプト、ツール呼び出しの1行サマリー（編集diffの統計を含む）、最終応答のみが表示されます。複雑なマルチステップ作業で中間過程のビジュアルノイズを削減し、結果に集中できます。

```bash
# NO_FLICKERモードを有効化（必須前提）
export CLAUDE_CODE_NO_FLICKER=1

# 実行後、Ctrl+OでFocus Viewをトグル
claude
```

[GitHub v2.1.97](https://github.com/anthropics/claude-code/releases/tag/v2.1.97)

### Claude Managed Agents APIリリース（4/9）

Anthropicが大規模エージェントをプログラマティックにデプロイできる**Managed Agents** APIプラットフォームをリリースしました。エージェントの構成、デプロイ、モニタリングを単一APIで処理できるコンポーザブルなプラットフォームです。Notion、楽天、Asana、Sentryが初期導入事例として参加しており、セッションあたり$0.08/時間の価格設定です。

[Anthropicブログ](https://claude.com/blog/claude-managed-agents)

## 開発者ワークフローティップス

### agent-skills — Addy OsmaniのプロダクショングレードなAIコーディングスキル集

Google Cloud AIディレクターのAddy Osmaniが、19個のプロダクショングレードなエンジニアリングスキルをオープンソースで公開しました。7つのスラッシュコマンドで構成され、Hyrum's Law（「観測可能なすべての振る舞いに誰かが依存する」）やChesterton's Fence（「理由を理解する前に削除するな」）といったGoogleエンジニアリング原則がAIエージェントのワークフローに組み込まれています。

- **構成**: 7つのスラッシュコマンド、19個のスキル
- **特徴**: AIが「技術的には動作するがプロダクションで問題になる」コードを生成するのを防止
- **インストール**: `git clone https://github.com/addyosmani/agent-skills`

```bash
# Claude Codeでスラッシュコマンドとして使用
/review    # コードレビュー
/plan      # 実装計画
/test      # テスト戦略
```

[GitHub](https://github.com/addyosmani/agent-skills)

### Awesome Design.MD — AIにデザインシステムを伝える標準フォーマット

60以上のサービス（Claude、Vercel、Stripe、Notion、Spotify）のデザインシステムをMarkdownで整理したコレクションです。プロジェクトルートに`DESIGN.md`を置くと、AIエージェントがそのサービスのデザイン言語を参照して一貫したUIを生成します。色、タイポグラフィ、スペーシング、コンポーネントなど9つの標準化されたセクションで構成されています。

CLAUDE.mdがコーディングルールをAIに伝えるなら、DESIGN.mdはデザインルールを伝える役割です。

[GitHub](https://github.com/VoltAgent/awesome-design-md)

## コミュニティニュース

- **macOS TCP 49日時限爆弾バグを発見**: XNUカーネルで正確に49日17時間後にTCPネットワーキングが完全に停止するバグが発見されました。`tcp_now`カウンターの32ビット整数オーバーフローが原因で、唯一の解決策はしきい値前の再起動です。長期稼働のMacサーバーを使用する開発者は定期再起動のスケジュールを確認してください。[ソース](https://photon.codes/blog/we-found-a-ticking-time-bomb-in-macos-tcp-networking)

- **「マルチエージェントオーケストレーションはなぜうまくいかないのか？」**: $5,000を投じてマルチエージェントフレームワークをテストした結果、トークンコストが単一エージェントの5〜10倍であるにもかかわらず、生産性向上はそれに比例しないとの分析です。コンテキスト同期、エージェント間通信のオーバーヘッド、検証コストという3つの構造的ボトルネックを特定しています。[ソース](https://shalomeir.substack.com/p/multi-agent-orchestration-problems)

- **Addy Osmaniの「コードエージェントオーケストラ」**: マルチエージェントコーディングの実践パターンを分析しています。サブエージェント、共有タスクリストベースのエージェントチーム、階層的委任の3パターンを比較し、「ボトルネックがコード生成から検証に移動した」という核心的な洞察を提示しています。[ソース](https://addyosmani.com/blog/code-agent-orchestra/)

## 知っておくと便利な小さな変更点

- **`refreshInterval`ステータスライン設定**: ステータスラインコマンドをN秒ごとに自動再実行する設定が追加されました
- **`/agents`に`N running`表示**: 実行中のサブエージェント数がエージェントタイプの横に表示されます
- **Cedarポリシーファイルのシンタックスハイライト**: `.cedar`、`.cedarpolicy`ファイルに対応
- **MCP HTTP/SSEメモリリーク修正**: サーバー再接続時に約50MB/hrの未解放バッファが蓄積されていた問題を修正
- **429リトライ指数バックオフ**: 従来約13秒ですべてのリトライを消費していた問題が修正され、指数バックオフが最小値として適用されます
- **サブエージェントworktreeディレクトリリーク修正**: worktree分離のサブエージェントの作業ディレクトリが親セッションのBashツールにリークしていた問題を修正
- **Context-low警告方式の変更**: 永続行ではなく一時的なフッター通知として表示されるようになりました
- **BashツールOTELトレーシング**: トレーシング有効時にサブプロセスがW3C `TRACEPARENT`環境変数を継承します

### 1. Claude AI、イラン軍事作戦での使用が確認 —「ブラックリスト発表の数時間後に」（3/3〜5）

## 開発者ワークフローティップス

### Simon Willisonの「Agentic Engineering Patterns」— AIコーディング時代のデザインパターン

Simon Willison（Django共同創設者、datasette開発者）がClaude Code、Codex等のAIコーディングエージェントと効果的に協業するためのパターンガイドを連載しています。1994年のGoFデザインパターン本にインスピレーションを受け、エージェンティックコーディングで繰り返し発生する問題への具体的な解決策を提示しています。

**主要な原則：**
- 初期コード生成コストは急落しましたが、**品質の高いコードは依然として高コスト** — 検証とレビューへの投資が必要です
- Red/Green TDDがエージェント時代に**必須**になりました — テストがエージェントのコードを客観的に検証する唯一の手段です
- 開発者の知識蓄積とパターン認識は代替不可能な競争力です

このガイドは「チャプター」形式で、公開後も継続的に更新される生きているドキュメントです。Hacker Newsでも大きな反響を得ています。

[Agentic Engineering Patterns](https://simonwillison.net/guides/agentic-engineering-patterns/) | [Hacker News](https://news.ycombinator.com/item?id=47243272)

### v2.1.69 プラグイン開発者向けの隠れた改善点（3/5）

v2.1.69のヘッドライン機能（Voice STT、/claude-api）はすでに紹介されましたが、プラグイン・スキル開発者に重要な変更が多数含まれています。

- **`${CLAUDE_SKILL_DIR}`**：スキルが自身のディレクトリを参照できる変数を追加 — 相対パスの問題を解決
- **`/reload-plugins`**：セッション再起動なしでプラグイン変更を即時反映
- **`InstructionsLoaded` フックイベント**：指示がロードされた時点でフックを実行可能に — 動的設定インジェクションが可能
- **フックイベントに `agent_id`、`agent_type`、`worktree` フィールド追加**：マルチエージェント・ワークツリー環境でのコンテキスト認識が向上
- **`pathPattern`**：マーケットプレイスソースで正規表現ベースのパスマッチングをサポート
- **`git-subdir` プラグインソースタイプ**：Gitリポジトリのサブディレクトリをプラグインソースとして指定可能

```bash
# プラグイン変更を即時反映
/reload-plugins

# スキル内から自身のディレクトリを参照
cat ${CLAUDE_SKILL_DIR}/templates/default.md
```

## セキュリティ・制限事項

### v2.1.69：.mcp.json信頼ダイアログの無音バイパス脆弱性を修正（3/5）

v2.1.69で、`.mcp.json`ファイルのMCPサーバーが信頼ダイアログで**ユーザーの同意なく自動的にすべて有効化**されていたセキュリティ脆弱性が修正されました。以前は信頼ダイアログが表示されましたが、個別サーバーの選択なくすべてのサーバーが一括承認される問題がありました。

この脆弱性は、Check Point Researchが公開したMCP自動承認バイパス攻撃ベクターと類似しています。まだアップデートしていない場合は、即座の適用を推奨します。

```bash
brew upgrade --cask claude-code
```

## 知っておくと便利な小さな変更点

- **v2.1.69 長時間セッションのメモリリーク大規模修正**：会話メッセージ、フックイベント、React Compilerキャッシュのメモリリークが修正されました。長時間セッションを頻繁に利用するユーザーはアップデート後にメモリ使用量の改善を実感できます。
- **v2.1.69 大容量バイナリコミット時の数GBメモリスパイク修正**：追跡されていない大容量バイナリファイルがある状態でのコミット時のメモリ急増が修正されました。
- **`sandbox.enableWeakerNetworkIsolation` 設定追加（macOS）**：カスタムMITMプロキシを使用する企業環境でのTLS検証問題を解決するサンドボックスネットワーク分離緩和オプションです。
- **`includeGitInstructions` 設定追加**：内蔵のコミット/PRワークフロー指示を削除できる設定です。CLAUDE.mdで独自のGitワークフローを定義している場合に便利です。
- **gitignoreされたディレクトリからのネストされたスキルロードを修正**：`node_modules`等から意図せずスキルがロードされていたセキュリティ問題が修正されました。

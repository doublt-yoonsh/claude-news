## 主要な新機能と実践活用

### Claude Code `/advisor` スラッシュコマンド — v2.1.101 で利用可能

v2.1.101 に追加された `/advisor` コマンドが注目を集めています。Anthropic の Advisor Strategy（4/9 発表）を Claude Code で直接使用できる機能です。

**動作方法**：
1. Claude Code で `/advisor` を実行し、モデルを選択（Sonnet 4.6 または Opus 4.6）
2. その後、Claude Code が複雑な判断、繰り返しエラー、タスク完了チェックポイントで自動的にアドバイザーモデルを呼び出し
3. アドバイザーは会話トランスクリプト全体（元のリクエスト、ツール呼び出し、ファイル読み取り、コマンド出力、推論過程）にアクセス
4. ただし、アドバイザーはファイルシステム、シェル、Web、MCP サーバーに直接アクセス不可

**推奨構成**：Sonnet 4.6 をメインモデル、Opus 4.6 をアドバイザーに設定すれば、コスト効率と重要な判断の推論深度を両立できます。

**パフォーマンス数値**：SWE-bench Multilingual で Sonnet + Opus アドバイザーが Sonnet 単独比 +2.7 ポイント（74.8% vs 72.1%）、エージェントタスクあたりコストは 11.9% 削減。

[Claude Blog](https://claude.com/blog/the-advisor-strategy) | [Advisor Tool Docs](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool)

## 開発者ワークフローのヒント

### `/advisor` を活用したコスト最適化ワークフロー

Claude Code で Opus 4.6 をデフォルトモデルとして使用しているチームなら、`/advisor` による逆転構成を試す価値があります：

```bash
# Claude Code で /advisor を実行し Opus 4.6 を選択
# → メインモデルは Sonnet 4.6、アドバイザーは Opus 4.6
# → 日常的なコーディングは Sonnet が処理、重要なアーキテクチャ判断時に Opus が助言

# 効果：
# - SWE-bench 基準のパフォーマンス低下はわずか 2.7pp
# - コスト 11.9% 削減
# - Haiku + Opus アドバイザーの組み合わせはコスト 85% 削減（ただしパフォーマンス低下大）
```

特に大規模コードベースでの反復的リファクタリングにおいて、Sonnet が基本実行を担当し Opus が「このリファクタリングの方向性は正しいか？」を判断するパターンがコストパフォーマンスに優れています。

[Claude Blog](https://claude.com/blog/the-advisor-strategy)

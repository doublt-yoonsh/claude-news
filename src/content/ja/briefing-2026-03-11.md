## 主要な新機能と実践活用

### v2.1.72 — effortレベル簡素化と開発者向けQoL改善 (3/10)

v2.1.72でeffortレベルが従来の4段階（low/medium/high/max）から**3段階（low/medium/high）**に簡素化されました。ステータスバーの表示も○（low）、◐（medium）、●（high）に変更され、現在の設定を直感的に確認できます。

**主な変更点：**

- **`/plan`即時進入**：`/plan fix the auth bug`のように説明を直接渡すと、即座にプランモードに入ります
- **`/copy`にファイル書き込み機能**：`w`キーで選択内容をクリップボードではなくファイルに直接保存 — SSH環境で特に便利です
- **`ExitWorktree`ツール追加**：worktreeセッションをツールで終了可能になりました
- **bash自動承認拡大**：`lsof`、`pgrep`、`tput`、`ss`、`fd`、`fdfind`が追加されました
- **バンドルサイズ約510KB削減**：全体的なロード速度が改善されました
- **音声認識の改善**：リポジトリ名や開発用語の認識精度が向上しました

```bash
# 即座にプランモードに入る
/plan fix the auth bug

# SSHでファイルに直接書き込み（クリップボード不要）
/copy  # → wキーでファイルに書き込み
```

## 開発者ワークフローティップス

### Amazon、AIコード変更にシニアエンジニアの承認を義務化 — 相次ぐ障害からの教訓

Amazonは、AIツールで生成・修正されたコードに対して**シニアエンジニアの事前承認を義務化**しました。3月5日にAmazon.comのショッピングサイトが6時間停止した障害が直接のきっかけであり、2025年12月のKiro AIアシスタントによる13時間のAWS障害などが累積した結果です。

内部文書では「新規導入されたGenAIユースケースには完全に確立されたセーフガードが不足している」と認めています。

**Claude Codeユーザーへの実践的教訓：**
- **AIコード生成速度 ≠ デプロイ準備完了**：デプロイパイプラインがAIのコード生産速度に対応できない可能性があります
- **「人間のフィルター」の重要性を再確認**：自動生成コードに対するシニアレベルの最終検証は必須です
- **テスト通過 ≠ 本番環境の安全性**：機能テストだけでなく、デプロイの安全性やロールバック計画まで検証が必要です

### Agent Safehouse — macOSでローカルAIエージェントを安全にサンドボックス化

ローカルで実行されるAIエージェントがシステムに意図しない変更を加えることを防止するmacOSネイティブのサンドボックスツールです。**deny-firstセキュリティモデル**により、明示的に許可されたディレクトリのみアクセス可能です。

**主な特徴：**
- Bashスクリプト1つでインストール、依存関係なし
- LLMベースの自動プロファイル生成で最小権限を設定
- SSHキー、AWSクレデンシャルなどの機密ファイルをカーネルレベルでブロック
- エージェントごとに独立したサンドボックス環境

Claude Codeの組み込みサンドボックスと併用して、追加のセキュリティレイヤーを構築したい開発者に有用です。

[agent-safehouse.dev](https://agent-safehouse.dev) | [GitHub](https://github.com/nicholasgriffintn/agent-safehouse)

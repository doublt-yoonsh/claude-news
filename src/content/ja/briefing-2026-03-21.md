## 主要な新機能と実践活用

### Claude Code v2.1.81 — スクリプト実行の最適化とモバイル権限リレー（3/20）

v2.1.80の翌日にリリースされた**v2.1.81**は、自動化シナリオとリモート権限管理に注力しています。

**主な変更：**
- **`--bare`フラグ** — フック、LSP、プラグイン同期、スキルディレクトリウォークをすべてスキップする軽量モード。`ANTHROPIC_API_KEY`または`--settings`経由の`apiKeyHelper`が必須。OAuth・キーチェーン認証と自動メモリは無効化。スクリプト/自動化呼び出しに最適
- **`--channels`権限リレー** — 権限機能を宣言したチャネルサーバーがツール承認プロンプトをスマートフォンに転送。リモートセッションでモバイルから権限承認が可能
- **OAuth同時セッション修正** — 複数のClaude Codeセッションが同時に実行される際、1つのセッションがOAuthトークンを更新すると他のセッションが再認証を要求する問題を解決
- **ボイスモード安定化** — リトライ失敗を無視して誤解を招く「ネットワークを確認してください」メッセージを表示する問題を修正。サーバーがWebSocket接続を静かに切断した際のオーディオ復旧問題を解決
- **`CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS`修正** — structured-outputsベータヘッダーを正しく抑制し、Vertex/Bedockプロキシゲートウェイでの400エラーを解消
- **Node.js 18クラッシュ修正** — Node.js 18環境でのクラッシュ問題を解決
- **Bashコマンドの不要な権限プロンプト修正** — 文字列にダッシュ(-)を含むBashコマンドでの不要な権限要求を解消
- **ワークツリーセッション復元** — ワークツリーでセッションを再開すると自動的にそのワークツリーに切り替え
- **MCPツール呼び出し表示改善** — MCP read/searchツール呼び出しが単一の「Queried {server}」行に折りたたみ表示（Ctrl+Oで展開）
- **`!` bashモードの発見性改善** — インタラクティブコマンド実行時にClaudeが`!`モードを提案
- **プラグイン最新性改善** — ref追跡プラグインが毎回ロード時に再クローンして上流の変更を反映
- **Remote Controlセッション改善** — タイトル自動生成、`/rename`同期修正、`/exit`アーカイブ安定化
- **MCP OAuth CIMD対応** — Dynamic Client Registrationのないサーバー向けClient ID Metadata Document（SEP-991）サポート
- **プランモード変更** — 「clear context」オプションをデフォルトで非表示に（`"showClearContextOnPlanAccept": true`で復元）
- **Windows行単位ストリーミング無効化** — レンダリング問題によりWindows（WSL含む）で行単位レスポンスストリーミングを無効化

## 開発者ワークフローティップス

### v2.1.81 `--bare` — 自動化スクリプトでClaude Codeを軽量実行

CI/CDパイプラインや自動化スクリプトでClaude Codeを呼び出す際、`--bare`フラグで不要な初期化をスキップできます。

```bash
ANTHROPIC_API_KEY=sk-... claude -p "このコードをレビューして" --bare
```

フック、LSP、プラグイン同期、スキルディレクトリウォークをすべてスキップし、高速でクリーンな実行が可能です。

### `--channels`権限リレーでモバイルから承認

リモートサーバーでClaude Codeを実行しながら、ツール承認をスマートフォンで受けることができます。チャネルサーバーが権限機能を宣言すると、承認プロンプトがモバイルに転送されます。

### MCPツール呼び出しの折りたたみ表示

v2.1.81でMCP read/searchツール呼び出しが単一行に折りたたまれ、出力がすっきりしました。詳細はCtrl+Oで展開可能。

## 知っておくと便利な小さな変更点

- **v2.1.81 プランモード「clear context」デフォルト非表示**: プラン承認後のコンテキストクリアオプションがデフォルトで非表示に。`"showClearContextOnPlanAccept": true`で復元可能。
- **v2.1.81 Windows行単位ストリーミング無効化**: レンダリング問題によりWindowsおよびWSLで行単位レスポンスストリーミングを無効化。
- **v2.1.81 Remote Controlセッションタイトル**: ジェネリックタイトルの代わりに最初のプロンプトベースでタイトルを自動生成。3番目のメッセージ後に更新。
- **v2.1.81 tmuxクリップボード問題修正**: 高速なCmd+Tab後のペーストでクリップボード内容が失われる問題を解決。
- **v2.1.81 `/btw`修正**: アクティブなレスポンス中に使用した際、ペーストしたテキストが含まれない問題を解決。
- **v2.1.81 透明フック添付ファイル**: 不可視のフック添付ファイルがトランスクリプトモードでメッセージ数を水増しする問題を解決。

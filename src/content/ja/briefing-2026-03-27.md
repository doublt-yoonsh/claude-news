## 主要な新機能と実践活用

### Anthropic vs. Pentagon — Lin 判事、予備的差止命令を発令（3/26）

Rita Lin 判事が、トランプ政権による Anthropic 製品の連邦使用禁止措置をブロックする**予備的差止命令（preliminary injunction）**を発令しました。今週毎日追跡してきたこの訴訟の、最初の実質的な結果です。

**判決の要点：**
- 政府の禁止措置が Anthropic の**合衆国憲法修正第1条で保護された表現**に対する処罰であるかを疑問視
- 命令は政府の控訴のため **7日間停止（stay）**
- 政府は即座に控訴する意向を表明

AI 企業が倫理的理由で軍事契約を拒否する権利について、業界全体の先例となる見通しです。

[Bloomberg](https://www.bloomberg.com/news/articles/2026-03-26/anthropic-wins-court-order-blocking-trump-ban-on-use-of-ai-tool) | [CBS News](https://www.cbsnews.com/news/anthropic-ruling-judge-trump-pentagon-ai/) | [Axios](https://www.axios.com/2026/03/26/judge-temporarily-blocks-pentagon-ban-anthropic)

### Auto Mode — Claude が自らパーミッションを判断する新モード（3/24）

Anthropic が **Auto Mode** をリサーチプレビューとしてリリースしました。クラシファイアが各ツール呼び出しの安全性をリアルタイムで評価し、ファイル書き込みや Bash コマンドを自動承認します。

**動作方式：**
- 安全な操作（ファイル読み取り、検索）：自動承認
- 危険な操作（大量削除、機密データアクセス、悪意のあるコード実行）：ユーザー確認を要求
- クラシファイアが各呼び出しを個別に評価

**対象：** Team プラン（Enterprise、API は後日対応）
**対応モデル：** Sonnet 4.6、Opus 4.6

[Anthropic Blog](https://claude.com/blog/auto-mode) | [TechCrunch](https://techcrunch.com/2026/03/24/anthropic-hands-claude-code-more-control-but-keeps-it-on-a-leash/)

### v2.1.84 — Windows ネイティブ PowerShell ツール（プレビュー）

Windows ユーザー向けの**ネイティブ PowerShell ツール**がオプトインプレビューとして追加されました。既存の Bash ツールと併用して Windows ネイティブコマンドの実行が可能です。

[PowerShell ツールドキュメント](https://code.claude.com/docs/en/tools-reference#powershell-tool)

### v2.1.85 — フック条件付き `if` フィルター

フックに `if` フィールドが追加され、パーミッションルール構文（`Bash(git *)` など）で実行条件を指定できるようになりました。すべてのツール呼び出しでフックプロセスを起動するオーバーヘッドを削減します。

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "if": "Bash(git *)",
        "command": "echo 'git コマンド検出'"
      }
    ]
  }
}
```

- **「Claude Code を作りながら学んだこと：Skills の使い方」**：Anthropic 内部の Skills 開発実務を公開。Skills はマークダウンだけでなくスクリプト+データを含むフォルダ構造で、Gotchas セクションや段階的開示ファイルシステム設計を紹介しています。今週最多の96ポイントを獲得。[原文スレッド](https://x.com/trq212/status/2033949937936085378)

- **「十分に詳細な仕様はコードである」**：「仕様書でコードを代替できる」というエージェンティックコーディングの主張に対する技術的批判。仕様が十分に精密になると、結局コードと同じ形に収束すること、OpenAI Symphony プロジェクトの SPEC.md で実際にバグが発生した事例を分析しています。[原文](https://haskellforall.com/2026/03/a-sufficiently-detailed-spec-is-code)

- **「余白を作る：少なくすることが偉大さを生む方法」**：すべてを1%ずつ改善しようとするのをやめれば、最も重要な一つを30%改善するエネルギーが確保されます。AI ツールで実行速度が上がるほど、「何をしないか」の判断がより重要になるという洞察。[原文](https://longform.asmartbear.com/focus/)

## 主要な新機能と実践活用

### Windows PowerShell ツール — opt-in プレビュー（v2.1.84）

Windows ユーザー待望のネイティブ PowerShell サポートが opt-in プレビューとして導入されました。WSL や Git Bash に頼ることなく、PowerShell を直接使用できます。

- Windows 環境でネイティブコマンド実行が可能
- opt-in 方式のため設定での有効化が必要
- Bash ツールと同じ権限モデルを適用

### 条件付きフック — `if` フィールド導入（v2.1.85）

フックに `if` フィールドが追加され、permission rule 構文で条件付き実行が可能になりました。すべてのツール呼び出しでフックプロセスをスポーンしていたオーバーヘッドが大幅に削減されます。

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "if": "tool == 'Bash'",
        "command": "echo 'Bash コマンド検出' >&2"
      }
    ]
  }
}
```

以前はすべてのツール呼び出しでフックが実行されていましたが、条件を指定して必要な場合のみトリガーできるようになりました。

### MCP マルチサーバー環境変数（v2.1.85）

`CLAUDE_CODE_MCP_SERVER_NAME` と `CLAUDE_CODE_MCP_SERVER_URL` 環境変数が追加され、複数の MCP サーバーを動的に構成できるようになりました。CI/CD やマルチ環境でコード変更なしに MCP サーバーを切り替えられます。

```bash
export CLAUDE_CODE_MCP_SERVER_NAME="staging-server"
export CLAUDE_CODE_MCP_SERVER_URL="http://localhost:3000"
```

## 開発者ワークフローティップス

### Anthropic の長時間エージェントハーネス設計パターン

Anthropic エンジニアリングブログで、長時間自律コーディングセッション向けの **3エージェントアーキテクチャ** が公開されました。

- **Planner**: タスク分解と実行計画の策定
- **Generator**: 実際のコード生成と修正
- **Evaluator**: 生成結果の品質評価とフィードバック

GAN にインスパイアされたこのパターンは、フロントエンドデザインの品質と自己評価の精度を同時に向上させます。マルチ時間のコーディングセッションで品質を維持しながら自律作業を続けるための核心的な設計手法です。

[Anthropic Engineering Blog](https://www.anthropic.com/engineering/harness-design-long-running-apps)

### Claude Code チートシート — 最新コマンドリファレンス

コミュニティ製の Claude Code チートシートが注目を集めています。最新バージョンのコマンド、ショートカット、設定、環境変数、MCP サーバー構成を一覧にまとめたリファレンスです。作業中のクイックルックアップに便利です。

[cc.storyfox.cz](https://cc.storyfox.cz)

### 並列 Worktree でコミット速度を最大化

「Being Productive with Claude Code」というポストで、並列 worktree システムがコミットスループットを劇的に向上させた実践事例が共有されました。

- 反復作業とビルド待ち時間を自動化で除去
- 複数の worktree で真の並列開発を実現
- AI が単純なコード生成を超え、開発者生産性の「乗数効果」を発揮

[neilkakkar.com](https://neilkakkar.com/productive-with-claude-code.html)

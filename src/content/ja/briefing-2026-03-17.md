## 主要な新機能と実践活用

### Claude Code v2.1.77 — 本日リリース、Opus 4.6出力トークンが64kデフォルトに（3/17）

本日リリースされた**v2.1.77**の主要な変更点です。

**主な変更：**
- **Opus 4.6のデフォルト最大出力トークンが64kに引き上げ**（上限：128k）— 大規模コード生成、リファクタリング、ドキュメント作成時の出力打ち切り問題が大幅に改善
- **`allowRead`サンドボックス設定** — `denyRead`領域内で特定パスの読み取りアクセスを再許可。モノレポで機密ディレクトリをブロックしつつ一部設定ファイルのみ許可する場合に有用
- **`/copy N`コマンド** — N番目に新しいアシスタントの応答をクリップボードにコピー。従来の`/copy`は最新の応答のみ対応
- **「Always Allow」複合bashコマンドバグ修正** — `&&`で連結したコマンドに「Always Allow」を適用すると過剰に広い権限ルールが保存される問題を修正
- **オートアップデーターのメモリ蓄積修正** — バイナリダウンロードの重複によるメモリ蓄積問題を解決
- **PreToolUseフックのセキュリティ修正** — `"allow"`を返すフックが`deny`権限ルールをバイパスするバグを修正
- **CRLFファイル上書き時の改行変換バグ修正** — Windows環境プロジェクトでの安定性向上
- **ターミナルUI改善** — vimモード、ハイパーリンク、tmuxサポート強化

### v2.1.77 `allowRead`の活用例

モノレポで`node_modules`をブロックしつつ特定の設定ファイルのみ許可したい場合：

```json
{
  "permissions": {
    "denyRead": ["node_modules/"],
    "allowRead": ["node_modules/.cache/config.json"]
  }
}
```

## 知っておくと便利な小さな変更点

- **v2.1.77 `/copy N`**: 最新ではなくN番目の過去の応答をクリップボードにコピー。複数の応答を比較・参照する際に便利。
- **v2.1.77 CRLF改行保持**: Writeツールがファイル上書き時にCRLF改行を変換しないよう修正。Windowsプロジェクト協業時の安定性向上。
- **v2.1.77 ターミナルvimモード＆tmux**: vimキーバインディングとtmux環境でのハイパーリンクレンダリング改善。
- **v2.1.77 オートアップデーターメモリ修正**: 長時間実行時にバイナリダウンロードの重複によりメモリが蓄積する問題を解決。

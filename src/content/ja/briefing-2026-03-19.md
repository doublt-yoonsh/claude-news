## 主要な新機能と実践活用

### Claude Code v2.1.79 — リモートコントロールと安定性向上（3/18）

v2.1.78に続き同日リリースされた**v2.1.79**は、開発者体験（DX）と安定性に焦点を当てています。

**主な変更：**
- **`--console`認証フラグ** — `claude auth login --console`でAnthropic Console（API課金）認証を直接実行可能。チーム・エンタープライズ環境での認証ワークフロー簡素化
- **ターン所要時間表示** — `/config`メニューに「Show turn duration」トグル追加。各ターンの所要時間をリアルタイム確認
- **起動時メモリ使用量〜18MB削減** — 全シナリオでメモリ効率が向上
- **非ストリーミングAPIフォールバック改善** — 試行あたり2分のタイムアウト追加でセッションの無限待機を防止
- **`CLAUDE_CODE_PLUGIN_SEED_DIR`マルチパス対応** — Unixでは`:`、Windowsでは`;`で複数シードディレクトリを指定可能
- **`claude -p`サブプロセスハング修正** — Python `subprocess.run`等で明示的stdinなしで呼び出し時のハングを解決
- **Ctrl+C `-p`モード修正** — printモードでCtrl+Cが正常動作
- **`/btw`ストリーミング中応答修正** — ストリーミング中の`/btw`がメインエージェント出力ではなくサイド質問に正しく回答
- **`SessionEnd`フック修正** — インタラクティブ`/resume`でセッション切替時に`SessionEnd`フックが正常発火
- **ボイスモード起動修正** — `voiceEnabled: true`設定時の起動でボイスモードが正常に有効化

**VSCode専用：**
- **`/remote-control`** — セッションをclaude.ai/codeにブリッジし、ブラウザやスマホから作業を継続可能
- **AI生成セッションタブタイトル** — 最初のメッセージを基にセッションタブタイトルを自動生成
- **Thinking pill修正** — 応答完了後「Thinking」の代わりに「Thought for Ns」を表示
- **サイドバーセッションdiffボタン修正**

[GitHub Releases](https://github.com/anthropics/claude-code/releases)

## 開発者ワークフローティップス

### v2.1.79 `/remote-control` — ブラウザからセッション継続

VSCodeで作業中のClaude Codeセッションをclaude.ai/codeにブリッジして、外出先でスマホや他のデバイスから作業を継続できます。

```
# VSCode Claude Codeパネルで
/remote-control
```

### `--console`フラグでAPI認証を簡素化

チーム環境でAnthropic Console（API課金）認証をCLIから直接実行：

```bash
claude auth login --console
```

### ターン所要時間の確認

`/config` → 「Show turn duration」を有効化 → 各ターンの所要時間をリアルタイムで確認し、プロンプト最適化に活用。

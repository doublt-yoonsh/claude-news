## 開発者ワークフローティップス

### コンテキスト60%閾値ルール — /compact より /clear が効果的

複数の実務者が独立して同じ結論に到達しました：**コンテキスト使用量を60%以下に維持**すべきです。自動圧縮（auto-compaction）は約83.5%で発動しますが、詳細の20〜30%しか保持しない損失圧縮です。

**実践パターン**：
1. 現在の計画と進捗をマークダウンファイルに保存
2. `/clear` でコンテキストを完全リセット
3. 新しいセッションでそのファイルを読み込んで再開

`/compact` よりこの方法が効果的な理由は、次のセッションにどの情報を引き継ぐかをユーザーが直接制御できるためです。

```bash
# 進捗をマークダウンに保存してリセット
claude "現在の計画と進捗を progress.md に保存して"
/clear
claude "progress.md を読んで続きを進めて"
```

[Medium - Effective Claude Code Workflows](https://medium.com/@sean.j.moran/effective-claude-code-workflows-in-2026-what-changed-and-what-works-now-c93ebc6f8f50)

### 4段階分離ワークフロー — Research → Plan → Implement → Validate

2026年4月時点でコミュニティで最も広く採用されているパターンです。各段階間でコンテキストを整理し、1つのセッションですべてを処理しないことが核心です。

**計画段階の具体的プロセス**：
1. Claude に「まだ実装せず、計画だけ立てて」とリクエスト
2. エディタで計画を開き、間違っている箇所にコメントを追加
3. 「すべてのコメントを反映して、まだ実装しないで」と再送信
4. 最終承認後にのみ実装開始

この方法は、事前計画なしの実装では成功率が約33%に留まるという Anthropic 内部テストの結果と一致しています。

## セキュリティ・制限事項

### Pro プラン Claude Code アクセス除外 A/B テスト騒動 (4/22)

Anthropic が新規 Claude Pro（$20/月）加入者から Claude Code アクセスを除外する A/B テストを実施し、大きな反発を招きました。サポートドキュメントが「Pro or Max」から「Max」のみに変更され、料金ページもグローバルに更新されました。

**経緯**：
- Anthropic 成長担当 Amol Avasare：「新規 prosumer 加入者の約2%を対象とした小規模テスト」
- 既存の Pro/Max 加入者には影響なし
- しかし公開料金ページとサポートドキュメントは全体的に変更され混乱が発生
- その後、変更はロールバック

**背景**：サブスクリプション料金が実際のトークンコストの1/10程度になるケースがあり、利用パターンの変化に伴うプラン再設計の圧力があるとみられています。Simon Willison は「非常に混乱する状況」と評価しました。

[The Register](https://www.theregister.com/2026/04/22/anthropic_removes_claude_code_pro/) | [Simon Willison](https://simonwillison.net/2026/Apr/22/claude-code-confusion/) | [GeekNews](https://news.hada.io/topic?id=28762)

### Mythos モデル不正アクセス事件 — サードパーティ契約業者経由 (4/21)

Anthropic のサイバーセキュリティ特化モデル Claude Mythos Preview への不正アクセス事件が発生しました。Bloomberg が最初に報道しました。

**侵入経緯**：
- Discord コミュニティの非公開グループが Mythos 発表当日にアクセスに成功
- Anthropic の他モデルの URL パターンを分析して Mythos のオンライン位置を推測
- サードパーティ契約業者の従業員1名がアクセス提供に関与
- グループの目的はサイバーセキュリティではなく「新しいモデルを試すこと」

**Anthropic の対応**：調査中であり、自社システムへの影響の証拠はないと表明。報告された活動はサードパーティベンダー環境の外に拡大していないとしています。

Bloomberg が「公開するには危険すぎるモデル」と報道した Mythos が発表当日に非公式アクセスを許したことは、限定プレビュー配布のセキュリティ管理に疑問を投げかけています。

## 知っておくと便利な小さな変更点

- **Skills で繰り返しワークフローを自動化**：`~/.claude/skills/` に保存したパーソナルスキルは全プロジェクトで利用可能。コンテキストがマッチすると Claude が自動適用するため、毎セッションの設定説明が不要に
- **サブエージェントでメインコンテキストを保護**：`.claude/agents/` に専門サブエージェントを定義すると、別のコンテキストウィンドウで作業し圧縮された要約のみメインセッションに返却。大規模コードベース探索時のコンテキスト汚染防止に効果的
- **公式プラグインマーケットプレイスの拡大**：`claude-plugins-official` リポジトリに Supabase、Firebase、Discord、Telegram などのサードパーティ統合が追加。「Anthropic Verified」バッジは手動の品質・安全レビューを通過したプラグインに付与

## 主要な新機能と実践活用

### v2.1.73 — modelOverrides、/simplify・/batchバンドルコマンド、HTTPフック (3/11)

v2.1.73で`modelOverrides`設定が追加され、モデルピッカーのエントリをカスタムプロバイダーのモデルID（例：Bedrock推論プロファイルARN）にマッピングできるようになりました。自社モデルエンドポイントを使用するエンタープライズ開発者に特に有用です。

**主な変更点：**

- **`/simplify`バンドルコマンド**：変更されたコードを自動レビューし、再利用性・品質・効率性をチェックして改善します
- **`/batch`バンドルコマンド**：複数のタスクを一括処理できるバッチコマンドが追加されました
- **HTTPフック対応**：シェルコマンドの代わりにURLにJSONをPOSTしてJSON応答を受け取るHTTPベースのフックを設定できます
- **worktree間設定共有**：同一リポジトリのgit worktree間でプロジェクト設定とauto memoryが共有されます
- **VS Codeセッション管理改善**：アクティビティバーにスパークアイコンが追加され、すべてのClaude Codeセッションをリスト表示しエディタとして開けます
- **プランマークダウンビュー**：VS Codeでプランを完全なマークダウンドキュメントとして表示し、コメントでフィードバックを残せます
- **Opus 4.6デフォルトeffort変更**：Max/TeamサブスクライバーのOpus 4.6デフォルトeffortがmediumに設定されます

```bash
# modelOverridesの例（settings.json）
# "modelOverrides": { "opus": "arn:aws:bedrock:..." }

# コード品質を自動改善
/simplify

# 複数タスクを一括実行
/batch
```

## 開発者ワークフローティップス

### Claude 3/11障害の事後分析 — 開発者が学ぶべき依存関係管理の教訓

3月11日、Claude AIが世界規模で約2時間の障害を経験しました。Downdetectorでは**10,000人以上**が問題を報告し、Claude Codeは**OAuth認証障害**により開発者が作業中にロックアウトされました。ただし、Claude APIは正常に動作していました。

**開発者への教訓：**

- **API vs. Web/CLIの分離を認識する**：Claude APIは影響を受けませんでしたが、Claude.aiとClaude Code CLIがダウンしました。プロダクション環境ではAPI直接呼び出しの方が安定しています
- **ローカルフォールバック戦略**：Claude Codeがダウンしても作業を継続できるよう、git stash、ローカルメモ、代替ツール戦略を準備しておきましょう
- **クラウド単一依存のリスク**：エンタープライズ環境ではAIコーディングツールの障害に備えたワークフロー設計が必要です

### MCP、月間SDKダウンロード9,700万件突破 — AIツール統合のデファクトスタンダードに

Anthropicが2024年11月に公開した**Model Context Protocol（MCP）**が月間SDKダウンロード**9,700万件**を突破し、AIツール統合のデファクトスタンダードとしての地位を確立しました。Claude、ChatGPT、Cursor、Gemini、Microsoft Copilot、VS Codeなど主要クライアントがすべてMCPをサポートしています。

2025年12月にAnthropicがMCPを**Linux Foundation傘下のAgentic AI Foundation（AAIF）**に寄贈して以降、Google、Microsoft、AWS、Cloudflare、Bloombergなどが共同支援しています。10,000以上のアクティブサーバーが稼働中で、「AIのUSB-C」と評されるほど急速に普及しています。

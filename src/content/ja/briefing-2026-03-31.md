## 主要な新機能と実践活用

### フリッカーフリーレンダリング — `CLAUDE_CODE_NO_FLICKER=1`（v2.1.88）

長時間セッションでの画面のちらつき問題を解決する新しいレンダリングモードが追加されました。仮想化されたスクロールバックとalt-screenレンダリングにより、フリッカーを完全に排除します。

```bash
# 環境変数で有効化
export CLAUDE_CODE_NO_FLICKER=1
```

[GitHub v2.1.88](https://github.com/anthropics/claude-code/releases)

### PermissionDeniedフック（v2.1.88）

Autoモードでクラシファイアがコマンドを拒否した際に発火する新しいフックイベントです。`{retry: true}`を返すことでリトライを許可できます。拒否されたコマンドは通知として表示され、`/permissions` Recentタブで確認可能です。

```json
{
  "hooks": {
    "PermissionDenied": [
      {
        "command": "echo '拒否コマンドをログ' >> /tmp/denied.log"
      }
    ]
  }
}
```

### 名前付きサブエージェントの@メンション対応（v2.1.88）

名前が付けられたサブエージェントが`@`メンションの候補リストに表示されるようになりました。チームエージェントを名前で直接参照してメッセージを送信でき、マルチエージェントワークフローがより直感的になりました。

### Claude Codeの隠れた15の強力機能

Claude Codeの創設者Boris Chernyが、あまり知られていない15の機能をまとめて話題になっています。

- **`--teleport`**：セッションを別のマシンに移動
- **`/loop` & `/schedule`**：繰り返しタスクの自動化とcronスケジューリング
- **`/branch`**：セッションを分岐して代替案を探索
- **`/btw`**：メインタスクを中断せずにサイドクエリ
- **`/batch`**：複数エージェントの並列実行
- **`--bare`**：10倍速いスクリプトモード起動
- **`--add-dir`**：複数リポジトリの同時作業
- **`/voice`**：音声でコーディング指示
- **Chrome拡張**：ブラウザからClaude Codeセッションを管理

## 開発者ワークフローティップス

### Claude Codeでセキュリティコードレビューを実施する

セキュリティ専門企業SpecterOpsのAndrew Luke氏が、Claude Codeをアプリケーションセキュリティ評価の「フォースマルチプライヤー」として活用する方法を公開しました。

**主要テクニック：**
- 攻撃者視点のセキュリティ重視システムプロンプト設計
- デーモンシステムマッピング、Cypherクエリ処理の追跡、セキュリティコントロールの特定に活用
- プライベートコードはローカルデプロイまたはAmazon Bedrockの使用を推奨 — 公開Claude.aiの使用は非推奨

```markdown
# セキュリティレビュー用CLAUDE.mdの例
- すべてのユーザー入力パスを追跡し、サニタイゼーションの有無を確認
- SQLクエリ、システムコマンド、ファイルパスでのインジェクション可能性を優先的にレビュー
- 認証・認可バイパス経路を体系的に探索
```

[SpecterOps Blog](https://specterops.io/blog/2026/03/26/leveling-up-secure-code-reviews-with-claude-code/)

### コンテキスト管理が生産性の鍵

実戦プロジェクトに基づくベストプラクティスガイドで、コンテキストを最も重要なリソースとして強調しています。

- **セッション分離**：タスク単位で新しいセッションを開始し、無関係なタスク間で`/clear`を使用
- **70%ルール**：コンテキスト使用量が70%に達したら先制的に`/compact`を実行
- **リサーチの委任**：探索的な作業はサブエージェントに委任してメインコンテキストを保護
- **フック > CLAUDE.md**：毎回必ず実行すべきタスク（フォーマット、リント、セキュリティチェック）はCLAUDE.mdではなくフックで設定 — フックは決定的（100%）、CLAUDE.mdは助言的

[Ran The Builder](https://ranthebuilder.cloud/blog/claude-code-best-practices-lessons-from-real-projects/) | [MorphLLM](https://www.morphllm.com/claude-code-best-practices)

### v2.1.88 安定性の大規模改善 — 30件以上のバグ修正

長時間セッションの安定性とメモリ関連の問題が集中的に解決されました。

- **プロンプトキャッシュミス**：ツールスキーマバイトの変更による長時間セッションのキャッシュミスを修正
- **CLAUDE.md重複注入**：長いセッションでネストされたCLAUDE.mdが数十回再注入される問題を修正
- **StructuredOutputキャッシュバグ**：複数スキーマ使用時の約50%失敗率を修正
- **メモリリーク**：大きなJSON入力がLRUキャッシュキーとして保持されるリークを修正
- **OOMクラッシュ**：1GiB以上のファイルでEditツール使用時のOOMを防止
- **レートリミットメッセージ**：「Rate limit reached」の代わりに実際の権限エラーメッセージを表示

## コミュニティニュース

- **「AIコーディングエージェントがフリーソフトウェアを再び重要にする」**：AIエージェントが非開発者でもコードを間接的に修正できるようにすることで、フリーソフトウェアの「4つの自由」に実質的な意味を回復できるという分析です。「私のAIエージェントがこのソフトウェアを修正できるか」が新しい購入基準になり得ます。[gjlondon.com](https://gjlondon.com)

- **Claude Codeがgit reset --hardを実行している？**：macOSユーザーがコミットされていない変更が10分ごとに削除される現象を報告しました。調査の結果、Claude Codeではなくユーザー自身のローカル自動化ツール（GitPython）が原因でした。共有作業ディレクトリでのツール間帰属混乱の教訓的な事例です。[GitHub Issue #40710](https://github.com/anthropics/claude-code/issues/40710)

- **GitHub CopilotがPRに広告を挿入した事件**：Copilotが自社機能やRaycastのプロモーションコンテンツをPR説明に無断挿入しました。Microsoftは該当「tips」機能を無効化し、AIコーディングツールの透明性の問題を浮き彫りにしました。[Zach Manson's Blog](https://notes.zachmanson.com/copilot-edited-an-ad-into-my-pr/)

## 知っておくと便利な小さな変更点

- **Thinking Summary デフォルト無効化**：v2.1.88からインタラクティブセッションで思考サマリーがデフォルトで生成されなくなりました。復元：`showThinkingSummaries: true`
- **`!command` Bashモード**：空のプロンプトに`!command`を貼り付けると自動的にbashモードに切り替わります
- **CJK/絵文字ヒストリー修正**：日本語/中国語/韓国語や絵文字を含むプロンプト履歴が4KB境界で切り捨てられる問題が修正されました
- **`/stats`データ保存**：30日以降の履歴データ損失とサブエージェント/フォークのトークン漏れが修正されました
- **スクロールバック保持**：長いセッションで上にスクロールした際にスクロールバックが消える問題が修正されました
- **Windows改善**：Shift+Enterが改行ではなく送信される問題、PowerShellツールのstderr進捗の誤報が修正されました

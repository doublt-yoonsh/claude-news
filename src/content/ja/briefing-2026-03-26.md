## 開発者ワークフローティップス

### Claude Code v2.1.83 — エンタープライズ管理機能の強化

昨日リリースされたv2.1.83の主要変更点を実践的な観点で整理します。

**チーム/エンタープライズ管理：**
- `managed-settings.d/`ドロップインディレクトリ：複数チームが独立したポリシーフラグメントをアルファベット順にマージ
- `disableDeepLinkRegistration`：`claude-cli://`プロトコルハンドラー登録を防止
- `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1`：サブプロセスからAnthropic/クラウドプロバイダーの認証情報を除去

**開発者体験：**
- `CwdChanged`、`FileChanged`フックイベント：direnv等のリアクティブな環境管理が可能に
- `sandbox.failIfUnavailable`：サンドボックス起動不可時にエラーで終了（サンドボックスなし実行を防止）
- トランスクリプト検索機能を追加
- non-streamingフォールバックトークンキャップ21k→64k、タイムアウト120s→300sに拡大

```bash
# 新しいフックイベントの活用例
# settings.jsonでCwdChangedフックによるdirenv自動ロード
{
  "hooks": {
    "CwdChanged": ["direnv allow"]
  }
}
```

## 知っておくと便利な小さな変更点

- **v2.1.83セキュリティ強化**: `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1`環境変数でサブプロセスから認証情報を自動除去 — CI/CDパイプラインで有用。
- **MCP OAuthアップデート**: Client ID Metadata Documentサポート追加 — MCPサーバー認証プロセスの改善。
- **ref-trackedプラグインの更新方式変更**: 毎回ロード時に再クローンでプラグインの最新性を保証 — プラグインキャッシュ問題を解決。
- **Auto Modeリサーチプレビュー**: Claudeが安全な操作と危険な操作を自ら判断し、自動実行の可否を決定する新モード。

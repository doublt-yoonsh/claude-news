## 主要な新機能と実践活用

### Claude Code v2.1.78 — プラグイン永続ステートとセキュリティ強化（3/18）

昨夜リリースされた**v2.1.78**の主要な変更点です。

**主な変更：**
- **`StopFailure`フックイベント** — APIエラー（レートリミット、認証失敗等）でターンが終了した際に発火する新フック。自動化ワークフローでのエラー検知・復旧パイプライン構築に有用
- **`${CLAUDE_PLUGIN_DATA}`変数** — プラグインの永続ストレージディレクトリ。プラグインアップデート後もデータが保持され、プラグイン開発者にとって大きな利便性向上
- **プラグインエージェントフロントマター拡張** — `effort`、`maxTurns`、`disallowedTools`をプラグイン配布エージェント定義で使用可能に
- **tmuxターミナル通知改善** — tmuxセッション内で実行時、外部ターミナルまで通知が伝達
- **行単位レスポンスストリーミング** — 生成されるごとに1行ずつ画面に出力
- **セキュリティ：サンドボックス依存関係欠如時のサイレント無効化修正** — 以前はサンドボックスが静かに無効化されていたが、起動時に警告を表示するよう修正
- **`.git`、`.claude`保護ディレクトリの書き込み権限修正** — プロンプトなしで書き込み可能だった問題を解決
- **`--worktree`フラグのスキル・フック未読み込み修正** — ワークツリーモードでスキルとフックが正常にロード
- **ボイスモード修正** — WSL2/WSLg互換性、修飾キーコンビネーションのプッシュトゥトーク修正

[GitHub Releases](https://github.com/anthropics/claude-code/releases)

### v2.1.78 `StopFailure`フックの活用例

APIレートリミット時に自動でSlack通知を送るフック設定：

```json
{
  "hooks": {
    "StopFailure": [
      {
        "command": "curl -X POST $SLACK_WEBHOOK -d '{\"text\": \"Claude Code APIエラー発生：確認が必要です\"}'",
        "timeout": 5000
      }
    ]
  }
}
```

### `${CLAUDE_PLUGIN_DATA}` プラグイン永続ステートの活用

プラグイン開発者は`${CLAUDE_PLUGIN_DATA}`を使って設定、キャッシュ、ユーザーデータを永続的に保存できます。プラグインアップデート時にもデータが保持されます。

### v2.1.78 セキュリティ修正

- **サンドボックス依存関係欠如時のサイレント無効化修正** — 以前は依存関係がない場合サンドボックスが静かに無効化されていたが、起動時に警告を表示。セキュリティに敏感な環境ではアップデート必須
- **`.git`、`.claude`保護ディレクトリ書き込み修正** — プロンプトなしで保護ディレクトリに書き込み可能だった問題を解決
- **`deny: ["mcp__servername"]` MCPサーバーツール除去修正** — denyルールがMCPサーバーツールを正常に除去できなかった問題を修正

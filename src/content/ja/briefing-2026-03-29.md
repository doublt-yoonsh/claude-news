## 主要な新機能と実践活用

### Cowork Dispatchメッセージ配信バグが解決（v2.1.87 + Desktop 1.1.9493）

Claude Desktop 1.1.9310でDispatchセッションが応答しなくなる深刻なバグが発見・修正されました。

**問題の概要：**
- モバイル/デスクトップからDispatchにメッセージを送信すると、受信・処理はされるが応答が返らない
- デスクトップバックエンドは正常動作するが、Cloud Control Relay（CCR）とクライアント間のリレー層で問題が発生
- GitHubに複数のバグレポート（#40178、#40179、#40257、#40283）が投稿

**解決策：**
- Claude Code v2.1.87でメッセージ配信ロジックを修正
- Claude Desktop 1.1.9493へのアップデートが必要

```bash
# Claude Codeのアップデート
brew upgrade --cask claude-code

# Claude Desktopのバージョン確認
# 設定 > アップデートで1.1.9493以上を確認
```

[GitHub v2.1.87](https://github.com/anthropics/claude-code/releases) | [GitHub Issue #40283](https://github.com/anthropics/claude-code/issues/40283)

## おすすめコラム＆読み物

- **「平均ユーザー向けにデザインするな」**: Jakob Nielsenが、デジタル世界のユーザー行動は正規分布ではなくべき乗則（power law）に従うと分析しています。上位5%のパワーユーザーが売上の80%を生み出すため、「平均」に合わせた設計は両方のユーザーを失望させます。Progressive Disclosureで両極のユーザーを満足させる戦略が鍵です。[UX Tigers](https://www.uxtigers.com/post/p50-vs-p95)

- **「見積もりはなぜ失敗するのか（それでもなぜ必要なのか）」**: ソフトウェアの見積もりは「Complex（複雑系）」領域に属し、どんなに経験豊富でも正確な予測が本質的に不可能です。しかし外部契約、チーム間の依存関係、ROI計算などの理由で依然として必要です。核心は、見積もり（estimate）、計画（plan）、約束（commitment）を分離することです。[leadership.garden](https://leadership.garden)

## 注目プロジェクト＆ツール

- **31 LLM交差検証自律株式売買システム**: 31のLLMエージェントを6段階リレーパイプラインで構成した完全自律の株式売買システムです。マクロ分析（7エージェント）、セクタースクリーニング（10）、企業分析（8）、最終CIO決定レイヤーで構成されます。Validatorエージェントがハルシネーション（存在しない銘柄の推奨）を防止し、Red-teamingで処分効果やサンクコストバイアスを矯正します。実口座（1,000万ウォン、約110万円）で運用中です。[Telegram](https://t.me/K_Agent_Alpha)

- **VibeCodeStats**: バイブコーディングツールの使用統計をリアルタイムで可視化するダッシュボードです。開発者がどのAIコーディングツールを使用しているかを確認できます。[vibecodingstats.com](https://vibecodingstats.com/)

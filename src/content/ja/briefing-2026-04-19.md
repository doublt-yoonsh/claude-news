## 開発者ワークフローティップス

### CLAUDE.md最適化 — 200行以内、約100個の指示バジェット

Claude Code公式ベストプラクティスガイドで、CLAUDE.md作成の定量的基準が整理されました。

**重要な数値**：
- **システムプロンプトが約50個の指示を消費** → プロジェクトに割り当て可能な指示は**約100個**
- **200行以内**を維持 — 長くなるほど指示の遵守率が低下
- **コンテキスト使用率90%以上**で出力品質が低下

**実践パターン**：
- コードのコピーではなく`src/auth.ts:42`形式の**file:lineポインタ**で参照
- ドメイン知識はCLAUDE.mdではなく**スキルに分離**して必要時のみロード
- ビルド・テストコマンドとコアコンベンションを最上部に簡潔に配置

```markdown
# 良い例 — 簡潔なCLAUDE.md
## ビルド
- `pnpm dev` — 開発サーバー
- `pnpm test` — テスト実行

## コンベンション
- 新しいAPI: src/routes/api.ts:15 のパターンを参照
- エラー処理: src/lib/errors.ts を参照
```

「指示を追加するたびに、他の指示の遵守率がわずかに低下する」という点を覚えておく必要があります。CLAUDE.mdが肥大化すると、実質的に使用可能なコンテキストが減少する逆効果が生じます。

[Claude Code Best Practices](https://code.claude.com/docs/en/best-practices)

### Contexty — AIコンテキストを可視化・制御するオープンソースツール

AIコーディングツールがどのコンテキストを参照しているかを開発者が直接確認・制御できるVS Code拡張機能です。GeekNewsで3ポイント。

- **リアルタイム可視化**：AIに渡されるコンテキストをVS Codeでモニタリング
- **アーキテクチャアンチパターン検出**：問題のあるコンテキスト構成を自動検知
- **自動コンテキスト圧縮**：不要なコンテキストを削減しトークン節約
- **細かいアクセス制御**：AIツールのファイルアクセス範囲を制御

Opus 4.7のトークン消費増加と相まって、コンテキスト最適化がコスト管理の中核となっています。

[GitHub](https://github.com/ttalkkak-lab/opencode-contexty)

## 主要な新機能と実践活用

### DOJ、Anthropicの修正第1条主張を正面から反論 — 40ページの書面を提出（3/17、今週の核心）

司法省（DOJ）がカリフォルニア連邦裁判所に**40ページの反論書面**を提出し、Anthropicの仮処分申請の却下を求めました。3/24審理を前に法的攻防が最高潮に達しています。

**DOJの核心的主張：**
- **「行為であり表現ではない」**: Anthropicが政府契約条件を拒否したのは修正第1条で保護される「表現」ではなく「行為」であると主張
- **「報復動機があっても結果は同じ」**: たとえ報復的動機があったとしても、国家安全保障上の懸念という正当な根拠のみで同一の措置を取ったと主張
- **「利用規約が受け入れ不可に」**: Anthropicが自律兵器と大規模監視への制限を主張した後、その利用規約が行政府にとって受け入れ不可能になったと明示

### Pentagon CTO、「Anthropicなしの生活にかなり自信あり」と発言（3/18）

Pentagon次官（CTO）Emil Michaelが、Anthropic代替について**公に自信を表明**しました。

**主要発言：**
- **「すでにOpenAIを展開済み」**: 「ここ数週間ですでにOpenAIを展開し、Geminiから始めて他のモデルも展開予定」
- **「ワークフローは非常に類似」**: 競合モデル間のワークフローが類似しているため、移行の混乱は最小限と主張
- **「単一プロバイダー依存が問題」**: 「1つの主要プロバイダーに依存することは国防省には適さない」
- **モデル別の強みを認識**: Claude=コーディング、xAI=リアルタイムコンテンツ、Gemini=YouTube統合に強みがあると言及
- **180日の移行期限**: 防衛企業もDoD業務でClaudeを使用していないことを認証する必要あり

## 開発者ワークフローティップス

### Claude Code Channels — 5分でTelegramから操作

v2.1.80でリサーチプレビューとして導入されたChannelsを活用すると、TelegramやDiscordからClaude Codeセッションに直接メッセージを送信できます。

```bash
# チャネルサポート付きでClaude Codeを起動
claude --channels
```

MCP基盤なのでコミュニティがSlack、WhatsAppなどの追加コネクタを独自に構築可能。v2.1.81の`--channels`権限リレーでモバイルからのツール承認も可能です。

### `--bare`モード — CI/CD自動化の要

```bash
ANTHROPIC_API_KEY=sk-... claude -p "このPRをレビューして" --bare
```

フック、LSP、プラグイン同期、スキルディレクトリウォークをすべてスキップし、自動化環境で高速かつクリーンな実行が可能です。

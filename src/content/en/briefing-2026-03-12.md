## New Features & Practical Usage

### v2.1.73 — modelOverrides, /simplify·/batch Bundled Commands, HTTP Hooks (3/11)

v2.1.73 adds a `modelOverrides` setting that maps model picker entries to custom provider model IDs (e.g., Bedrock inference profile ARNs) — particularly useful for enterprise developers using their own model endpoints.

**Key changes:**

- **`/simplify` bundled command**: Automatically reviews changed code for reuse, quality, and efficiency, then fixes issues found
- **`/batch` bundled command**: Batch-process multiple tasks in a single command
- **HTTP hooks**: Configure hooks that POST JSON to a URL and receive JSON responses instead of running shell commands
- **Shared configs across worktrees**: Project configs and auto memory are now shared across git worktrees of the same repository
- **VS Code session management**: New spark icon in the activity bar lists all Claude Code sessions, opening them as full editors
- **Plan markdown view**: View plans as full markdown documents in VS Code with support for inline comments as feedback
- **Opus 4.6 default effort change**: Medium effort is now the default for Max and Team subscribers on Opus 4.6

```bash
# modelOverrides example (settings.json)
# "modelOverrides": { "opus": "arn:aws:bedrock:..." }

# Auto-improve code quality
/simplify

# Batch-process multiple tasks
/batch
```

## Developer Workflow Tips

### Claude 3/11 Outage Post-Mortem — Dependency Management Lessons for Developers

Claude AI experienced a worldwide outage on March 11, lasting approximately 2 hours. Over **10,000 users** reported issues on Downdetector. Claude Code suffered an **OAuth authentication outage**, locking developers out mid-work — while the Claude API remained operational.

**Lessons for developers:**

- **API vs. Web/CLI separation**: The Claude API was unaffected while Claude.ai and Claude Code CLI went down. For production workflows, direct API calls provide better reliability
- **Local fallback strategies**: Maintain git stash habits, local notes, and alternative tool strategies so you can continue working when Claude Code is unavailable
- **Single-dependency risk**: Enterprise environments need workflow designs that account for AI coding tool outages

### MCP Surpasses 97 Million Monthly SDK Downloads — The De Facto Standard for AI Tool Integration

Anthropic's **Model Context Protocol (MCP)**, launched in November 2024, has crossed **97 million monthly SDK downloads** with 10,000+ active servers, establishing itself as the de facto standard for connecting AI to external tools. Every major client now supports MCP: Claude, ChatGPT, Cursor, Gemini, Microsoft Copilot, and VS Code.

After Anthropic donated MCP to the **Agentic AI Foundation (AAIF)** under the Linux Foundation in December 2025, co-founded with Block and OpenAI, with support from Google, Microsoft, AWS, Cloudflare, and Bloomberg, MCP has been called "the USB-C of AI."

## Security & Limitations

### Anthropic Exposes Industrial-Scale Model Theft by Chinese AI Labs

Anthropic has published evidence that three Chinese AI labs — **DeepSeek, Moonshot AI, and MiniMax** — conducted **industrial-scale distillation** of Claude using **24,000 fraudulent accounts** and over **16 million interactions**.

**Key details:**

- **DeepSeek**: 150,000+ exchanges focused on reasoning tasks, RL reward model grading, and censorship-safe rewrites of politically sensitive queries
- **Moonshot**: 3.4 million+ exchanges targeting agentic reasoning, tool use, coding, data analysis, and computer vision
- **"Hydra clusters"**: A single proxy setup controlled 20,000+ fraudulent accounts simultaneously, mixing extraction traffic with normal usage to avoid detection

Anthropic has deployed **behavioral fingerprinting** and chain-of-thought extraction pattern detection systems to strengthen defenses.

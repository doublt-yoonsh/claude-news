### Auto Mode — Claude Makes Its Own Permission Decisions (3/24)

Anthropic launched **Auto Mode** as a research preview. A classifier evaluates each tool call for safety, automatically approving file writes and Bash commands it deems safe while prompting the user for risky operations.

**How it works:**
- Safe operations (file reads, searches): auto-approved
- Dangerous operations (mass deletion, sensitive data access, malicious code): user confirmation required
- A real-time classifier evaluates each tool call individually

**Availability:** Team plan now; Enterprise and API coming soon
**Supported models:** Sonnet 4.6, Opus 4.6

[Anthropic Blog](https://claude.com/blog/auto-mode) | [TechCrunch](https://techcrunch.com/2026/03/24/anthropic-hands-claude-code-more-control-but-keeps-it-on-a-leash/)

### v2.1.84 — Native PowerShell Tool for Windows (Preview)

A **native PowerShell tool** is now available as an opt-in preview for Windows users. It runs alongside the existing Bash tool, enabling native Windows command execution.

[PowerShell tool docs](https://code.claude.com/docs/en/tools-reference#powershell-tool)

### v2.1.85 — Conditional `if` Field for Hooks

Hooks now support an `if` field using permission rule syntax (e.g., `Bash(git *)`) to control when they fire. This reduces process spawning overhead by skipping hooks that don't match the current tool call.

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "if": "Bash(git *)",
        "command": "echo 'git command detected'"
      }
    ]
  }
}
```

## Security & Limitations

### ShadowPrompt — Zero-Click XSS in Claude Chrome Extension (Patched)

A **zero-click XSS prompt injection** vulnerability was found in the Claude Chrome Extension. Any website could silently inject prompts without user interaction. Patched in extension version **1.0.41**.

**Action required:** Update your Chrome extension to 1.0.41 or later.

[The Hacker News](https://thehackernews.com/2026/03/claude-extension-flaw-enabled-zero.html)

### Claude Haiku 3 — End of Life on April 19

Claude Haiku 3 will be **retired on April 19, 2026**. If you're using this model, migrate to **Haiku 4.5** before the deadline.

## Minor Changes Worth Knowing

- **Token display change**: Counts ≥1M now show as "1.5m" instead of "1512.6k" (v2.1.84).
- **MCP description 2KB cap**: Prevents context bloat from OpenAPI-generated servers (v2.1.84).
- **Idle-return `/clear` nudge**: After 75+ minutes inactive, a prompt suggests `/clear` to avoid unnecessary token re-caching (v2.1.84).
- **Global system prompt caching**: Now works with `ToolSearch` enabled, including for users with MCP tools configured (v2.1.84).
- **VSCode rate limit banner**: Shows usage percentage and reset time (v2.1.84).
- **New environment variables**: `CLAUDE_CODE_MCP_SERVER_NAME`/`URL` (v2.1.85), `ANTHROPIC_DEFAULT_{OPUS,SONNET,HAIKU}_MODEL_SUPPORTS`/`_MODEL_NAME`/`_DESCRIPTION` (v2.1.84), `CLAUDE_STREAM_IDLE_TIMEOUT_MS` (v2.1.84).

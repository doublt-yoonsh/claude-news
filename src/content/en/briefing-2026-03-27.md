## Developer Workflow Tips

### Claude Code v2.1.85 — Conditional Hooks & Performance

Two releases shipped in a single day (v2.1.84 and v2.1.85).

**v2.1.85 highlights (3/26):**
- **Conditional hooks**: New `if` field using permission rule syntax (e.g., `Bash(git *)`) — hooks fire only on matching tool calls, reducing process spawn overhead
- **MCP multi-server**: `CLAUDE_CODE_MCP_SERVER_NAME` and `CLAUDE_CODE_MCP_SERVER_URL` env vars let `headersHelper` scripts distinguish between servers
- **Transcript timestamps**: Timestamp markers when `/loop` or `CronCreate` tasks fire
- **Scroll performance**: Pure TypeScript replaces WASM yoga-layout — smoother rendering for large transcripts
- **MCP OAuth**: RFC 9728 Protected Resource Metadata discovery support
- **Bug fixes**: `/compact` "context exceeded" error, `--worktree` in non-git repos, remote session memory leak, persistent ECONNRESET errors, multi-monitor `switch_display`, terminal stuck in enhanced keyboard mode (Ghostty/Kitty/WezTerm)

```jsonc
// v2.1.85 conditional hook example — only fires on git commands
{
  "hooks": {
    "PreToolUse": [
      {
        "if": "Bash(git *)",
        "command": "echo 'Git command detected'"
      }
    ]
  }
}
```

**v2.1.84 highlights (3/26):**
- **PowerShell tool for Windows** (opt-in preview): First native Windows shell tool
- **Model capability override**: `ANTHROPIC_DEFAULT_{OPUS,SONNET,HAIKU}_MODEL_SUPPORTS` env vars for pinned models on Bedrock/Vertex/Foundry
- **Idle-return prompt**: After 75+ minutes away, nudges `/clear`
- **`x-client-request-id` header**: Added to API requests for debugging
- **VSCode**: Rate limit warning banner showing usage % and reset time

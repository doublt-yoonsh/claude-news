## 주요 신규 기능 & 실전 활용

### v2.1.120 회수 & 안정화 기간 — 프로덕션 환경은 v2.1.117 고정 권장 (4/24)

v2.1.119와 v2.1.120이 24시간 간격으로 릴리스된 후, **8건의 리그레션이 동시에 보고**되면서 v2.1.120은 자동 업데이트에서 회수되었습니다. 이는 Claude Code 역사상 단일 릴리스 사이클에서 가장 많은 리그레션이 동시 보고된 사례 중 하나입니다.

**Critical/High 리그레션**:
- `--resume`/`--continue` 크래시 (Critical)
- 1M 컨텍스트로 사일런트 모델 스왑 (High)
- `CLAUDE.md` 무시 리그레션 (High)
- 자동 업데이트 자체 장애 (High)

**Medium 리그레션**:
- WSL2에서 `/mcp` 메뉴 프리즈
- macOS worktree 생성 행
- 터미널 리사이즈 시 UI 중복
- `sandbox.excludedCommands` 네트워크 차단 강제

```bash
# 프로덕션 환경에서 안정 버전 고정
npm install -g @anthropic-ai/claude-code@2.1.117

# 자동 업데이트 차단 (CI/CD 환경)
export DISABLE_UPDATES=1

# 현재 버전 확인
claude --version
```

현재 Anthropic이 수정 작업 중이며, 안정화될 때까지 v2.1.117 고정이 권장됩니다. v2.1.119는 대부분의 환경에서 동작하지만, WSL2나 worktree를 사용하는 경우 주의가 필요합니다.

## 개발자 워크플로우 팁

### "처음부터 알았으면 좋았을 Claude Code 팁" — Marmelab 실전 가이드

Marmelab의 개발팀이 Claude Code를 실전에 도입하면서 축적한 팁을 공개했습니다.

**증분 요청(Incremental Requests)**: 한 번에 전체 기능을 요청하지 말고, 한 단계씩 요청합니다. 리뷰 가능성이 유지되고 Claude의 집중도가 높아집니다.

**Preview를 품질 관리 도구로 활용**: Preview 도구를 통해 Claude가 사용자 관점에서 변경사항을 직접 확인하고 자기 수정할 수 있습니다. 사람이 발견하기 전에 문제를 잡을 수 있습니다.

**세션 종료 시 학습 문서화**: 세션을 마칠 때 "What did you learn?"을 질문하고, 결과를 CLAUDE.md, 스킬 파일, ADR에 반영합니다. 세션 간 기관 지식(institutional knowledge)이 축적됩니다.

**병렬 과부하 경고**: 배치 명령과 worktree를 과도하게 사용하면 세밀한 제어가 불가능해집니다. 에러가 사람이 잡을 수 있는 속도보다 빠르게 복합됩니다. "인간 병목은 버그가 아니라 기능이었다"는 통찰이 핵심입니다.

[Marmelab](https://marmelab.com/blog/2026/04/24/claude-code-tips-i-wish-id-had-from-day-one.html)

### `@filepath` 직접 참조로 검색 비용 절약

프롬프트에서 `@filepath`를 사용하면 Claude가 파일을 검색하지 않고 직접 참조합니다. 검색 도구 호출이 줄어들어 토큰을 절약하고 응답 속도가 빨라집니다.

```bash
# 검색 없이 직접 참조
claude "@src/auth/middleware.ts 의 토큰 검증 로직을 수정해줘"

# vs. 검색이 필요한 방식
claude "토큰 검증 로직을 수정해줘"  # → Glob/Grep 호출 발생
```

400K 토큰을 넘으면 관련성이 급격히 떨어지므로, 일반 컨텍스트로 작업의 95%를 처리하는 것이 권장됩니다. Extended context는 정말 필요한 경우에만 사용하십시오.

[Marmelab](https://marmelab.com/blog/2026/04/24/claude-code-tips-i-wish-id-had-from-day-one.html)

**권장 조치**: 안정적인 프로덕션 환경이 필요하면 v2.1.117로 고정하고, `DISABLE_UPDATES=1`로 자동 업데이트를 차단하십시오.

## 알아두면 좋은 소소한 변경사항

- **v2.1.120 자동 롤백**: v2.1.120으로 업데이트된 사용자는 자동 업데이트가 v2.1.119로 다운그레이드
- **`--agent <name>` permissionMode 적용**: 빌트인 에이전트의 `permissionMode`가 `--agent`로 실행 시에도 적용
- **`owner/repo#N` 링크 호스트 자동 감지**: git remote의 호스트를 사용하여 GitHub Enterprise 등에서도 올바른 링크 생성
- **`claude plugin tag`**: 플러그인 릴리스 시 버전 검증과 함께 git 태그 자동 생성
- **OpenTelemetry 개선**: `tool_result`와 `tool_decision` 이벤트에 `tool_use_id` 포함으로 도구 호출 추적 강화

## 주요 신규 기능 & 실전 활용

### v2.1.118 — Vim Visual Mode, Hook MCP 통합, 커스텀 테마 (4/23)

v2.1.118에서 여러 주요 기능이 추가되었습니다.

**Vim Visual Mode**: `v`(visual mode)와 `V`(visual-line mode)가 추가되어, 선택·오퍼레이터·시각 피드백이 지원됩니다. `j`/`k`는 NORMAL 모드에서 히스토리 탐색과 footer pill 선택에 사용됩니다.

**Hook에서 MCP 도구 직접 호출**: Hook의 `type`에 `"mcp_tool"`이 추가되어, Hook이 MCP 서버의 도구를 직접 실행할 수 있습니다. 기존에 Bash 명령으로 우회하던 MCP 연동을 Hook 수준에서 네이티브로 처리할 수 있게 되었습니다.

**`/cost`와 `/stats` 통합 → `/usage`**: 기존의 비용 확인(`/cost`)과 통계(`/stats`) 명령이 `/usage` 하나로 통합되었습니다.

**커스텀 테마 생성·전환**: `/theme`에서 이름을 지정한 커스텀 테마를 생성하고 전환할 수 있습니다.

```bash
# Vim visual mode 사용
# v → 선택 시작, d/y 등 오퍼레이터 적용

# Hook에서 MCP 도구 직접 호출 (settings.json)
# "type": "mcp_tool" 로 MCP 서버 도구를 hook에서 실행

# 통합된 사용량 확인
/usage

# 커스텀 테마 생성
/theme  # → 이름 지정하여 저장·전환
```

## 개발자 워크플로우 팁

### 테스트를 "외부 진실 소스"로 활용하기

테스트 없이 작업하면 Claude는 자신의 판단으로 결과를 검증합니다. 그런데 컨텍스트가 채워질수록 이 자기 검증의 정확도가 떨어집니다. 테스트는 세션 전체에서 정확하게 유지되는 **외부 진실 소스(external source of truth)**를 만듭니다.

**실전 적용**:
- 구현 전에 실패하는 테스트를 먼저 작성하도록 지시
- 각 red-to-green 사이클이 Claude에게 명확한 피드백 제공
- 컨텍스트가 90%에 도달해도 테스트 결과는 정확하게 유지

```bash
# 테스트 우선 워크플로우
claude "이 기능의 테스트를 먼저 작성해줘. 아직 구현하지 마."
# → 테스트 실패 확인 후
claude "테스트가 통과하도록 구현해줘."
```

[Claude Code Best Practices](https://code.claude.com/docs/en/best-practices)

### 반복 작업은 Skill로 만들기 — "주 1회 이상이면 스킬"

주 1회 이상 반복하면서 일관된 출력을 요구하는 작업은 Skill로 만들어야 합니다. 콘텐츠 포맷, 코드 리뷰 패턴, 리서치 요약 구조, 아웃리치 작성 등 반복 가능한 패턴이 대상입니다.

**Skill의 장점**: CLAUDE.md에 넣으면 매 세션마다 컨텍스트를 소비하지만, Skill은 컨텍스트가 매칭될 때만 로드됩니다. CLAUDE.md의 ~100개 지시사항 예산을 절약하면서 도메인별 전문 지식을 필요 시에만 활용할 수 있습니다.

```bash
# 개인 스킬 생성 (모든 프로젝트에서 사용)
mkdir -p ~/.claude/skills
# → code-review.md, pr-template.md 등 반복 패턴을 스킬로 저장
```

## 알아두면 좋은 소소한 변경사항

- **`/fork` 최적화**: 부모 대화 전체를 디스크에 쓰지 않고 포인터만 저장. 대규모 세션에서 fork 속도가 크게 개선
- **`DISABLE_UPDATES` 환경변수**: 모든 업데이트 경로를 차단. CI/CD 환경이나 관리형 배포에서 버전 고정에 활용
- **WSL에서 Windows 관리 설정 상속**: WSL 환경에서 Windows 측 managed settings가 자동으로 상속
- **Auto mode `"$defaults"` 지원**: `autoMode.allow`, `autoMode.soft_deny`, `autoMode.environment`에 `"$defaults"`를 포함하여 기본값 위에 커스텀 규칙 추가 가능
- **"Don't ask again" 옵션**: auto mode 진입 확인 프롬프트에서 "다시 묻지 않기" 선택 가능
- **MCP OAuth 수정 다수**: 자격증명 저장 충돌, 토큰 갱신, 인증 플로우 관련 여러 수정 포함

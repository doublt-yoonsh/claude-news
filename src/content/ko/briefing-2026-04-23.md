## 개발자 워크플로우 팁

### 컨텍스트 60% 임계값 규칙 — /compact보다 /clear가 낫다

여러 실무자가 독립적으로 동일한 결론에 도달했습니다: **컨텍스트가 60%를 넘지 않도록 관리**해야 합니다. 자동 압축(auto-compaction)은 약 83.5%에서 발동하지만, 세부 사항의 20~30%만 유지하는 손실 압축입니다.

**실전 패턴**:
1. 현재 계획과 진행 상황을 마크다운 파일에 저장
2. `/clear`로 컨텍스트를 완전 리셋
3. 새 세션에서 해당 파일을 읽으며 재시작

`/compact`보다 이 방식이 효과적인 이유는, 어떤 정보가 다음 세션에 전달될지 사용자가 직접 통제할 수 있기 때문입니다.

```bash
# 진행 상황을 마크다운에 저장 후 리셋
claude "현재 계획과 진행 상황을 progress.md에 저장해줘"
/clear
claude "progress.md를 읽고 이어서 진행해줘"
```

[Medium - Effective Claude Code Workflows](https://medium.com/@sean.j.moran/effective-claude-code-workflows-in-2026-what-changed-and-what-works-now-c93ebc6f8f50)

### 4단계 분리 워크플로우 — Research → Plan → Implement → Validate

2026년 4월 기준 커뮤니티에서 가장 많이 채택되는 패턴입니다. 각 단계 사이에서 컨텍스트를 정리하고, 한 세션에서 모든 작업을 몰아서 하지 않는 것이 핵심입니다.

**계획 단계의 구체적 프로세스**:
1. Claude에게 "구현하지 말고 계획만 세워줘"로 초안 요청
2. 에디터에서 계획을 열고 틀린 부분에 코멘트 추가
3. "모든 코멘트를 반영해줘, 아직 구현하지 마"로 재전달
4. 최종 승인 후에만 구현 시작

이 접근법은 Anthropic 내부 테스트에서 사전 계획 없는 구현의 성공률이 약 33%에 불과하다는 결과와 일치합니다.

## 보안/제한 이슈

### Pro 플랜 Claude Code 접근 제거 A/B 테스트 논란 (4/22)

Anthropic이 신규 Claude Pro($20/월) 구독자에 대해 Claude Code 접근을 제거하는 A/B 테스트를 실시하여 큰 논란이 일었습니다. 지원 문서가 "Pro or Max"에서 "Max"로 변경되었고, 가격 페이지에서도 Pro 플랜의 Claude Code 접근이 제거되었습니다.

**경과**:
- Anthropic 성장 담당 Amol Avasare: "신규 prosumer 가입자의 약 2%를 대상으로 하는 소규모 테스트"
- 기존 Pro/Max 구독자는 영향 없다고 밝힘
- 그러나 공개 가격 페이지와 지원 문서는 전체적으로 변경되어 혼란 발생
- 이후 변경사항이 롤백됨

**배경**: 구독 요금이 실제 토큰 소비량의 장부가 대비 1/10 수준인 경우가 있어, 사용 패턴 변화에 따른 요금제 재설계 압박이 있는 것으로 보입니다. Simon Willison은 "매우 혼란스러운 상황"이라고 평가했습니다.

[The Register](https://www.theregister.com/2026/04/22/anthropic_removes_claude_code_pro/) | [Simon Willison](https://simonwillison.net/2026/Apr/22/claude-code-confusion/) | [GeekNews](https://news.hada.io/topic?id=28762)

### Mythos 모델 무단 접근 사건 — 제3자 계약업체 경로 (4/21)

Anthropic의 사이버보안 특화 모델 Claude Mythos Preview에 무단 접근 사건이 발생했습니다. Bloomberg이 최초 보도했습니다.

**침입 경위**:
- Discord 커뮤니티의 비공개 그룹이 Mythos 발표 당일 접근에 성공
- Anthropic의 다른 모델 URL 형식을 분석하여 Mythos의 온라인 위치를 추측
- 제3자 계약업체 직원 1명이 접근 제공에 관여
- 그룹은 사이버보안 목적이 아닌 "새 모델을 시험해보는 것"에 관심

**Anthropic 대응**: 조사 중이며, 자사 시스템에 대한 영향 증거는 없다고 밝혔습니다. 보고된 활동이 제3자 벤더 환경 밖으로 확장되지 않았다고 합니다.

"공개하기에 너무 위험한 모델"로 Bloomberg이 보도한 Mythos가 발표 당일 비공식 접근을 허용한 것은, 제한적 프리뷰 배포의 보안 관리에 의문을 제기합니다.

## 알아두면 좋은 소소한 변경사항

- **Skills로 반복 워크플로우 자동화**: `~/.claude/skills/`에 저장한 개인 스킬은 모든 프로젝트에서 사용 가능. 컨텍스트가 매칭되면 Claude가 자동으로 적용하므로, 매 세션마다 선호사항을 재설명할 필요 없음
- **서브에이전트로 메인 컨텍스트 보호**: `.claude/agents/`에 전문 서브에이전트를 정의하면 별도 컨텍스트 윈도우에서 작업 후 압축된 요약만 메인 세션에 반환. 대규모 코드베이스 탐색 시 메인 세션의 컨텍스트 오염 방지에 효과적
- **플러그인 공식 마켓플레이스 확대**: `claude-plugins-official` 저장소에 Supabase, Firebase, Discord, Telegram 등 서드파티 통합이 추가. "Anthropic Verified" 배지는 수동 품질·안전 검토를 통과한 플러그인에 부여

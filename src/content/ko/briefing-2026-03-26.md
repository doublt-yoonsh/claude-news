## 개발자 워크플로우 팁

### Claude Code v2.1.83 — 엔터프라이즈 관리 기능 강화

어제 출시된 v2.1.83의 주요 변경사항을 실전 관점에서 정리합니다.

**팀/엔터프라이즈 관리:**
- `managed-settings.d/` 드롭인 디렉토리: 여러 팀이 독립적인 정책 파편을 알파벳순으로 머지
- `disableDeepLinkRegistration`: `claude-cli://` 프로토콜 핸들러 등록 방지
- `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1`: 서브프로세스에서 Anthropic/클라우드 자격증명 제거

**개발자 경험:**
- `CwdChanged`, `FileChanged` 훅 이벤트: direnv 등 반응형 환경 관리 가능
- `sandbox.failIfUnavailable`: 샌드박스 시작 불가 시 에러로 종료 (무샌드박스 실행 방지)
- 트랜스크립트 검색 기능 추가
- non-streaming 폴백 토큰 캡 21k→64k, 타임아웃 120s→300s로 확대

```bash
# 새 훅 이벤트 활용 예시
# settings.json에서 CwdChanged 훅으로 direnv 자동 로드
{
  "hooks": {
    "CwdChanged": ["direnv allow"]
  }
}
```

[GitHub Release](https://github.com/anthropics/claude-code/releases/tag/v2.1.83) | [Changelog](https://code.claude.com/docs/en/changelog)

## 알아두면 좋은 소소한 변경사항

- **v2.1.83 보안 강화**: `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1` 환경변수로 서브프로세스에서 자격증명 자동 제거 — CI/CD 파이프라인에서 유용.
- **MCP OAuth 업데이트**: Client ID Metadata Document 지원 추가 — MCP 서버 인증 프로세스 개선.
- **ref-tracked 플러그인 갱신 방식 변경**: 매 로드 시 재클론으로 플러그인 최신성 보장 — 플러그인 캐시 문제 해결.
- **Auto Mode 리서치 프리뷰**: Claude가 안전한 작업과 위험한 작업을 스스로 판단하여 자동 실행 여부를 결정하는 새 모드.

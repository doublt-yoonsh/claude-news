## 주요 신규 기능 & 실전 활용

### v2.1.74 — /context 최적화 가이드, autoMemoryDirectory, 메모리 누수 수정 (3/12)

v2.1.74에서 `/context` 명령이 대폭 개선되어 컨텍스트를 많이 차지하는 도구, 메모리 비대화, 용량 경고를 식별하고 **구체적인 최적화 팁**을 제안합니다. 컨텍스트 관리가 Claude Code 성능의 핵심인 만큼, 실전에서 즉시 활용할 수 있는 기능입니다.

**주요 변경사항:**

- **`autoMemoryDirectory` 설정 추가**: auto-memory 저장 디렉토리를 커스텀 경로로 지정할 수 있습니다. 팀 단위로 메모리를 공유하거나 프로젝트별로 분리할 때 유용합니다
- **메모리 누수 수정**: 스트리밍 API 응답 버퍼가 제너레이터 조기 종료 시 해제되지 않아 RSS 메모리가 무한 증가하던 문제가 수정되었습니다. npm 경로 사용자에게 특히 중요합니다
- **정책 우회 수정**: 관리 정책의 `ask` 규칙이 사용자 `allow` 규칙이나 스킬 `allowed-tools`로 우회되던 보안 문제가 수정되었습니다
- **Voice Mode macOS 수정**: 터미널에 마이크 권한이 없는 macOS 네이티브 바이너리에서 Voice Mode가 조용히 실패하던 문제가 수정되었습니다
- **RTL 텍스트 수정**: 히브리어, 아랍어 등 RTL 텍스트가 Windows Terminal, VS Code 통합 터미널에서 올바르게 렌더링됩니다
- **SessionEnd 훅 타임아웃 설정**: 종료 시 1.5초 후 강제 종료되던 SessionEnd 훅을 `CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS`로 조절할 수 있습니다

```bash
# auto-memory 디렉토리 커스텀 설정
# settings.json: "autoMemoryDirectory": "/path/to/shared/memory"

# 컨텍스트 최적화 진단
/context

# SessionEnd 훅 타임아웃 연장 (밀리초)
export CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS=10000
```

## 개발자 워크플로우 팁

### Pragmatic Engineer 설문: 개발자 95%가 AI를 주간 사용, Claude Code가 1위

Pragmatic Engineer가 3월 7일 발표한 **906명의 소프트웨어 엔지니어 대상 설문 조사**(경력 중앙값 11~15년, 주로 유럽·미국)에서 Claude Code가 AI 코딩 도구 1위를 차지했습니다.

**핵심 수치:**
- **95%**가 AI 도구를 주 1회 이상 사용, **75%**가 업무의 절반 이상에 AI 활용
- "가장 좋아하는 도구"로 **Claude Code 46%**, Cursor 19%, GitHub Copilot 9%
- 스타트업에서는 **75%**가 Claude Code를 주요 도구로 사용
- **55%**가 AI 에이전트를 정기적으로 사용 (18개월 전 거의 0%)
- 에이전트 사용자는 비사용자 대비 AI 도구에 대해 **긍정적 평가가 거의 2배** (61% vs 36%)

**개발자를 위한 실전 인사이트:**
- 2025년 5월 출시 후 **8개월 만에** GitHub Copilot과 Cursor를 추월 — 도구 전환의 속도가 전례 없이 빠릅니다
- 디렉터급 이상 시니어 리더가 Claude Code와 에이전트를 가장 적극적으로 사용 — AI 도구 숙련도가 커리어 경쟁력으로 직결되고 있습니다
- **70%가 2~4개 AI 도구를 동시에 사용** — 단일 도구에 의존하기보다 상황별 최적 도구 조합이 중요합니다

[Pragmatic Engineer](https://newsletter.pragmaticengineer.com/p/ai-tooling-2026) | [AI:Productivity](https://aiproductivity.ai/news/pragmatic-engineer-survey-ai-tooling-2026/)

### Anthropic × Mozilla — Claude가 Firefox에서 22개 보안 취약점 발견

Anthropic과 Mozilla가 협력하여 Claude Opus 4.6으로 Firefox 소스코드를 분석한 결과, 2주 만에 **22개의 보안 취약점**(CVE)을 발견했습니다. 이 중 **14개가 고위험(high-severity)**으로 분류되어, 2025년 전체 고위험 취약점 수정 건수의 약 20%에 해당합니다.

**기술적 세부사항:**
- 수천 개의 C++ 파일을 분석하여 **112건의 고유 리포트** 제출
- 발견된 취약점은 **Firefox 148.0**에 반영, 수억 명의 사용자에게 배포 완료
- 취약점 **발견과 패치 생성**에는 뛰어나지만, **작동하는 익스플로잇 작성** 성공률은 극히 낮음 (수백 번 시도 중 2건 성공, 샌드박스 비활성화 환경에서만)

**개발자를 위한 교훈:**
- AI는 현재 **공격보다 방어에 유리** — 취약점 탐지·패치 능력이 익스플로잇 작성 능력을 크게 앞섭니다
- 자체 프로젝트에서도 Claude Code를 활용한 보안 리뷰가 실효성이 있다는 실증적 사례입니다
- Claude Code Security(엔터프라이즈 프리뷰)의 기반이 되는 기술이기도 합니다

[Anthropic 공식](https://www.anthropic.com/news/mozilla-firefox-security) | [GeekNews](https://news.hada.io/topic?id=27272) | [TechCrunch](https://techcrunch.com/2026/03/06/anthropics-claude-found-22-vulnerabilities-in-firefox-over-two-weeks/)

Cloudflare가 Browser Rendering 서비스에 `/crawl` 엔드포인트를 퍼블릭 베타로 출시했습니다. 단 한 번의 API 호출로 전체 웹사이트를 탐색하고 콘텐츠를 수집할 수 있습니다.

**주요 기능:**
- HTML, Markdown, JSON 등 다양한 출력 포맷 지원
- depth 제한, URL 패턴 등 범위 제어
- 수정 타임스탬프 기반 증분 크롤링
- 브라우저 렌더링 없는 정적 모드
- robots.txt 준수

MCP 서버를 통해 Claude Code와 연동하면 웹 데이터 수집 워크플로우를 크게 단순화할 수 있습니다. 다만 커뮤니티에서는 "봇 차단과 스크래핑 서비스를 동시에 판매하는 아이러니"에 대한 논의도 있었습니다.

## 알아두면 좋은 소소한 변경사항

- **v2.1.74 에이전트 모델 ID 지원**: `claude-opus-4-5` 같은 전체 모델 ID가 에이전트 프런트매터 `model:` 필드와 `--agents` JSON 설정에서 올바르게 인식됩니다. 이전에는 조용히 무시되었습니다.
- **v2.1.74 MCP OAuth 수정**: 콜백 포트가 이미 사용 중일 때 인증이 중단되던 문제와, Slack 등 HTTP 200으로 에러를 반환하는 OAuth 서버에서 재인증 프롬프트가 표시되지 않던 문제가 수정되었습니다.
- **v2.1.74 플러그인 우선순위 변경**: `--plugin-dir`로 지정한 로컬 개발 플러그인이 동일 이름의 설치된 마켓플레이스 플러그인보다 우선합니다. 플러그인 개발 워크플로우가 개선되었습니다.
- **v2.1.74 미확인 슬래시 명령 경고**: 존재하지 않는 슬래시 명령에 인자를 전달하면 조용히 무시되는 대신 경고가 표시됩니다.
- **v2.1.74 Windows LSP 수정**: 잘못된 파일 URI로 인해 Windows에서 LSP 서버가 작동하지 않던 문제가 수정되었습니다.

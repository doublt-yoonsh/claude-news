## 주요 신규 기능 & 실전 활용

### Claude Code v2.1.80 — MCP 채널과 대규모 레포 최적화 (3/19)

v2.1.79 다음 날 릴리스된 **v2.1.80**은 MCP 확장성과 대규모 프로젝트 성능에 초점을 맞췄습니다.

**주요 변경:**
- **`--channels` (리서치 프리뷰)** — MCP 서버가 세션에 메시지를 푸시할 수 있는 새로운 채널 기능. 외부 이벤트(CI 완료, 알림 등)를 세션으로 직접 전달 가능
- **`rate_limits` 상태표시줄 필드** — 상태표시줄 스크립트에서 Claude.ai 사용량 한도(5시간·7일 윈도우, `used_percentage`, `resets_at`) 확인 가능
- **`source: 'settings'` 플러그인 소스** — `settings.json`에 플러그인 항목을 인라인 선언 가능. 별도 파일 없이 설정에서 직접 플러그인 등록
- **`effort` 프론트매터 지원** — 스킬과 슬래시 명령어에 `effort` 프론트매터를 추가해 모델 effort 레벨 오버라이드
- **대규모 레포 메모리 80MB 절감** — 25만 파일 규모 레포에서 시작 시 ~80MB 절약
- **`--resume` 병렬 도구 결과 누락 수정** — 병렬 도구 호출이 포함된 세션 복원 시 모든 tool_use/tool_result 쌍이 정상 복원
- **보이스 모드 WebSocket 수정** — Cloudflare 봇 탐지로 인한 비브라우저 TLS 핑거프린트 WebSocket 실패 해결
- **API 프록시 400 오류 수정** — Bedrock·Vertex 등 API 프록시에서 세밀한 도구 스트리밍 시 발생하던 400 오류 수정
- **`/remote-control` 비지원 환경 표시 수정** — 게이트웨이·서드파티 프로바이더에서 동작 불가능한 `/remote-control`이 노출되지 않도록 수정
- **`@` 파일 자동완성 개선** — 대규모 Git 레포에서 반응 속도 향상
- **`/effort` 표시 개선** — auto가 현재 어떤 레벨로 해석되는지 표시
- **플러그인 설치 팁 간소화** — 단일 `/plugin install` 명령어로 통합

[GitHub Releases](https://github.com/anthropics/claude-code/releases)

### Pentagon, Anthropic 외국인 인력 보안 우려 제기 (3/19)

Pentagon이 Anthropic의 **외국인 인력, 특히 중국 국적 직원**에 대한 새로운 보안 우려를 법원 서면에서 제기했습니다.

**핵심 사항:**
- **Pentagon 차관 Emil Michael**: "Anthropic은 다수의 외국 국적자를 고용하여 LLM 제품을 구축·지원하고 있으며, 중국(PRC) 국적자도 다수 포함"
- **PRC 국가정보법 리스크**: 중국 국적 직원이 PRC 국가정보법에 따라 정보를 제공할 경우 "적대적 리스크 수준이 증가"한다고 주장
- **업계 반박**: Foundation for American Innovation은 "업계 내에서 Anthropic은 외국 국적 직원의 내부 위협에 가장 진지하고 선제적으로 대응하는 것으로 널리 알려져 있다"고 반박
- **Anthropic의 실적**: Anthropic은 자체 플랫폼에서 AI를 이용한 최초의 중국발 사이버 스파이 캠페인을 탐지·차단하고, PRC를 서비스에서 차단한 전례 보유
- **3/24 심리 D-4**: 가처분 심리까지 4일

## 개발자 워크플로우 팁

### v2.1.80 `--channels` — MCP 서버에서 세션으로 메시지 푸시

리서치 프리뷰 단계의 새 기능으로, MCP 서버가 세션에 메시지를 직접 푸시할 수 있습니다. CI/CD 완료 알림, 외부 이벤트 등을 세션 내에서 실시간으로 수신 가능.

```bash
claude --channels
```

### `rate_limits`로 사용량 한도 모니터링

상태표시줄 스크립트에서 `rate_limits` 필드를 활용해 Claude.ai의 5시간·7일 사용량 한도와 리셋 시점을 표시할 수 있습니다.

### `effort` 프론트매터로 스킬별 모델 강도 조절

스킬이나 슬래시 명령어에 `effort` 프론트매터를 추가해 호출 시 모델 effort 레벨을 오버라이드. 간단한 작업은 낮은 effort, 복잡한 작업은 높은 effort로 최적화.

## 알아두면 좋은 소소한 변경사항

- **v2.1.80 대규모 레포 메모리 절감**: 25만 파일 레포에서 시작 시 ~80MB 절약. v2.1.79의 18MB에 이어 대폭 개선.
- **v2.1.80 `--resume` 병렬 도구 복원**: 병렬 tool_use/tool_result 쌍이 세션 복원 시 누락되지 않도록 수정.
- **v2.1.80 보이스 모드 WebSocket**: Cloudflare 봇 탐지 우회로 비브라우저 환경 WebSocket 안정화.
- **v2.1.80 API 프록시 호환성**: Bedrock·Vertex 등에서 세밀한 도구 스트리밍 400 오류 해결.
- **v2.1.80 `/effort` 표시 개선**: auto 모드가 실제로 어떤 effort 레벨로 해석되는지 확인 가능.
- **v2.1.80 managed settings 캐시 수정**: `remote-settings.json` 캐시 시 시작 시점에 관리 설정이 적용되지 않던 문제 해결.

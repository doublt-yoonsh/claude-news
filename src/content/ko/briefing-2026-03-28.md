## 개발자 워크플로우 팁

### Claude Code 창시자 Boris Cherny의 100줄 CLAUDE.md 워크플로우

Anthropic Staff Engineer이자 Claude Code 창시자인 Boris Cherny가 자신의 워크플로우를 공개하여 화제가 되고 있습니다. 커뮤니티에서 500~1,000줄 이상의 CLAUDE.md를 작성하는 것과 달리, Boris의 파일은 **약 100줄(~2,500 토큰)**이며 모든 줄이 실제 문제를 해결합니다.

**핵심 원칙:**
- **자기 개선 루프**: Claude가 실수할 때마다 `lessons.md`에 규칙을 추가하여 같은 실수를 반복하지 않도록 함
- **Plan Mode 필수**: 비자명한 작업은 반드시 Shift+Tab으로 Plan Mode 진입
- **서브에이전트 적극 활용**: 컨텍스트를 깨끗하게 유지
- **10~15 병렬 세션**: 터미널 5개 + 웹 5~10개 + 모바일, 각 세션마다 독립 git worktree
- **"Claude Code는 이제 100% Claude Code로 작성됨"**

[MindWired AI](https://mindwiredai.com/2026/03/25/claude-code-creator-workflow-claudemd/) | [VentureBeat](https://venturebeat.com/technology/the-creator-of-claude-code-just-revealed-his-workflow-and-developers-are)

### .claude/ 폴더 구조 완전 해부

`.claude/` 디렉토리의 구조와 역할을 분석한 글이 GeekNews에서 주목받고 있습니다.

- **CLAUDE.md**: 핵심 가이드라인 (200줄 이내 권장)
- **CLAUDE.local.md**: Git에 커밋하지 않는 개인 오버라이드
- **rules/**: 주제별 모듈화된 규칙
- **commands/**: 커스텀 슬래시 명령어
- **skills/**: 대화 컨텍스트 기반 자동 트리거 워크플로우
- **agents/**: 독립 시스템 프롬프트를 가진 서브에이전트

핵심 인사이트는 "최소 구성에서 시작하여 실제 필요에 따라 확장"하는 것입니다. 과도한 설정은 오히려 결과를 악화시킵니다.

## 주요 신규 기능 & 실전 활용

### v2.1.76 — MCP Elicitation 지원과 모노레포 최적화 (3/14 출시)

오늘 릴리스된 v2.1.76의 핵심은 **MCP Elicitation**입니다. MCP 서버가 작업 중 사용자에게 구조화된 입력을 요청할 수 있게 되었습니다 — 폼 필드나 브라우저 URL을 통한 인터랙티브 다이얼로그를 지원합니다.

**주요 변경:**
- **MCP Elicitation**: MCP 서버가 작업 도중 사용자 입력을 요청하는 인터랙티브 다이얼로그 지원. `Elicitation`/`ElicitationResult` 훅으로 응답 가로채기 및 오버라이드 가능
- **`-n` / `--name` 플래그**: 시작 시 세션 표시 이름 설정 — 여러 세션을 동시에 운영할 때 구분이 편리
- **`worktree.sparsePaths`**: 모노레포에서 git sparse-checkout 설정 지원 — 대규모 레포지토리의 워크트리 성능 대폭 개선
- **`/effort` 명령어**: effort 레벨을 슬래시 명령어로 직접 변경 가능
- **버그 수정**: deferred tools 압축 시 입력 스키마 소실, 슬래시 명령어 "Unknown skill" 표시, plan mode 재승인, 음성 모드 키입력 문제, Windows `/voice` 지원 수정

모노레포를 사용하는 팀이라면 `worktree.sparsePaths`가 특히 유용합니다. 필요한 경로만 체크아웃하여 워크트리 생성 속도와 디스크 사용량을 크게 줄일 수 있습니다.

### v2.1.75 — Opus 4.6 1M 컨텍스트 윈도우 기본 적용 (3/13)

Opus 4.6의 **1M 토큰 컨텍스트 윈도우**가 Max, Team, Enterprise 플랜에서 기본 활성화되었습니다. 기존 200K 대비 5배 확장이며, 동일 가격으로 제공됩니다.

대규모 코드베이스 분석, 긴 문서 참조, 복잡한 리팩토링 작업에서 컨텍스트 부족 문제가 크게 완화됩니다. `/color` 명령어로 프롬프트 바 색상을 설정하고, 세션 이름이 프롬프트 바에 표시되어 멀티 세션 환경에서의 가독성도 향상되었습니다.

### MCP SDK 다운로드 월 9,700만 회 돌파 — 사실상 업계 표준

Model Context Protocol(MCP) SDK 다운로드가 2026년 2월 기준 **월 9,700만 회**에 도달했습니다. Anthropic, OpenAI, Google, Microsoft, Amazon 등 모든 주요 AI 제공업체가 MCP를 지원하며, 사실상의 업계 표준으로 자리잡았습니다.

v2.1.76의 MCP Elicitation 지원과 맞물려, MCP 서버를 구축하거나 활용하는 것이 Claude Code 워크플로우 확장의 핵심 경로가 되고 있습니다.

### Vite 8.0 출시 — Rust 기반 Rolldown 번들러로 10~30배 빌드 속도 향상

**Vite 8.0**이 출시되며 기존 esbuild/Rollup 이중 구조를 Rust 기반 **Rolldown** 번들러로 통합했습니다(GeekNews). 빌드 속도가 **10~30배** 향상되었으며, 프론트엔드 프로젝트에서 Claude Code와 함께 빠른 개발-빌드-테스트 사이클을 구현하는 데 직접적인 이점이 있습니다.

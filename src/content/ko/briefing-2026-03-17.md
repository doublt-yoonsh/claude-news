## 주요 신규 기능 & 실전 활용

### Claude Code v2.1.77 — 오늘 릴리스, Opus 4.6 출력 토큰 64k 기본 적용 (3/17)

오늘 릴리스된 **v2.1.77**의 핵심 변경사항입니다.

**주요 변경:**
- **Opus 4.6 기본 최대 출력 토큰이 64k로 상향** (상한: 128k) — 대규모 코드 생성, 리팩토링, 문서 작성 시 출력 중단 문제가 크게 줄어듭니다
- **`allowRead` 샌드박스 설정** — `denyRead` 영역 내에서 특정 경로의 읽기 접근을 다시 허용. 모노레포에서 민감 디렉토리를 차단하면서 일부 설정 파일만 허용할 때 유용
- **`/copy N` 명령어** — N번째 최근 어시스턴트 응답을 클립보드에 복사. 기존 `/copy`는 가장 최근 응답만 복사했으나 이제 과거 응답도 선택 가능
- **"Always Allow" 복합 bash 명령어 버그 수정** — `&&`로 연결된 명령어에 "Always Allow"를 적용하면 지나치게 넓은 권한 규칙이 저장되던 문제 해결
- **자동 업데이터 메모리 누적 수정** — 바이너리 다운로드가 중복되며 메모리가 누적되던 문제 해결
- **PreToolUse 훅 `"allow"` 반환 시 `deny` 권한 규칙 우회 버그 수정** — 보안상 중요한 수정
- **CRLF 파일 덮어쓰기 시 줄바꿈 변환 버그 수정** — Windows 환경 프로젝트 작업 시 안정성 향상
- **터미널 UI 개선** — vim 모드, 하이퍼링크, tmux 지원 강화

[GitHub Releases](https://github.com/anthropics/claude-code/releases)

### v2.1.77 `allowRead` 활용 예시

모노레포에서 `node_modules`를 차단하되 특정 설정 파일은 허용하고 싶을 때:

```json
{
  "permissions": {
    "denyRead": ["node_modules/"],
    "allowRead": ["node_modules/.cache/config.json"]
  }
}
```

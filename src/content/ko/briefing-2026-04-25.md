## 주요 신규 기능 & 실전 활용

### v2.1.119 — /config 영구 저장, 멀티 플랫폼 PR 리뷰, Hook 실행 시간 추적 (4/23)

v2.1.118 직후 공개된 v2.1.119에서 여러 실용적 기능이 추가되었습니다.

**`/config` 설정 영구 저장**: `/config`에서 변경한 설정이 `~/.claude/settings.json`에 저장되며, project/local/policy 우선순위에 따라 오버라이드됩니다. 세션이 끝나도 설정이 유지됩니다.

**`prUrlTemplate` — 커스텀 코드 리뷰 URL**: 자체 코드 리뷰 시스템을 사용하는 팀을 위해 PR URL 템플릿을 커스터마이징할 수 있습니다.

**`--from-pr` 멀티 플랫폼 확장**: 기존 GitHub PR URL에 더해 **GitLab, Bitbucket, GitHub Enterprise** PR URL도 지원합니다. 멀티 플랫폼 환경에서 PR 기반 코드 리뷰가 가능해졌습니다.

**Hook `duration_ms` 추가**: `PostToolUse`와 `PostToolUseFailure` Hook에 도구 실행 시간(`duration_ms`)이 포함됩니다. 성능 병목 추적이나 느린 도구 감지에 활용할 수 있습니다.

```bash
# GitLab PR로 코드 리뷰 시작
claude --from-pr https://gitlab.com/org/repo/-/merge_requests/42

# Bitbucket PR도 지원
claude --from-pr https://bitbucket.org/org/repo/pull-requests/15

# Hook에서 도구 실행 시간 활용 (settings.json)
# PostToolUse hook 응답에 duration_ms 포함
```

**기타 개선**: PowerShell 도구 명령 자동 승인, `--print` 모드에서 에이전트 `tools:`/`disallowedTools:` 프론트매터 적용, Vim 모드에서 Esc가 INSERT 모드의 큐된 메시지를 되돌리지 않도록 수정, `CLAUDE_CODE_HIDE_CWD` 환경변수 추가.

## 개발자 워크플로우 팁

### 컨텍스트 품질이 effort 레벨보다 중요하다 — 80% 휴리스틱

"단순한 작업인데 Max effort를 써야 하는 것 같다"고 느끼면, **80%의 경우 문제는 effort가 아니라 상류 컨텍스트의 부재**입니다. BSWEN의 분석에 따르면, Low effort + 좋은 컨텍스트가 Max effort + 나쁜 컨텍스트보다 더 나은 결과를 보여줍니다.

**effort를 올리기 전 CLAUDE.md 점검 체크리스트**:
1. **Commands**: 빌드·테스트 명령이 복사-붙여넣기 가능한 형태인가
2. **Architecture**: 디렉토리 구조와 모듈 관계가 명시되어 있는가
3. **Patterns**: 핵심 컨벤션이 구체적 예시와 함께 있는가
4. **Gotchas**: 비직관적인 프로젝트 특이사항이 기록되어 있는가
5. **Currency**: 참조 정보가 현재 상태와 일치하는가

effort를 올리면 모델의 사고 예산이 증가하지만, 그 예산이 "프로젝트 상태 재구성"에 소비되면 실제 문제 해결에 남는 것이 없습니다. 컨텍스트를 먼저 개선하면 같은 effort에서 더 나은 결과를 얻을 수 있습니다.

[BSWEN](https://docs.bswen.com/blog/2026-04-19-claude-context-quality-vs-effort/)

### AI 코드 생성의 "과잉 편집" 문제 — Claude Opus 4.6이 가장 적게 수정

AI 모델에게 간단한 버그 수정을 요청하면, 실제 수정 범위를 넘어서 함수를 재구조화하고 변수를 리네이밍하고 헬퍼를 추가하는 "과잉 편집(over-editing)" 현상이 발생합니다. 이는 코드 리뷰를 어렵게 만들고, 테스트가 통과해도 불필요한 변경이 숨어 있게 됩니다.

연구 결과, **Claude Opus 4.6이 테스트된 모델 중 과잉 편집이 가장 적었습니다**(정규화 편집 거리 0.060). GPT-5.4는 0.395로 가장 높았습니다.

**실전 해결법**: 프롬프트에 "원본 코드의 구조와 로직을 최대한 보존하라"는 지시를 추가하면 모든 모델에서 과잉 편집이 크게 감소합니다.

```bash
# CLAUDE.md에 추가할 수 있는 지시
# "버그 수정 시 원본 코드의 구조와 로직을 최대한 보존할 것.
#  불필요한 리네이밍, 리팩토링, 헬퍼 함수 추가를 하지 말 것."
```

[원문](https://nrehiew.github.io/blog/minimal_editing/)

## 알아두면 좋은 소소한 변경사항

- **`CLAUDE_CODE_HIDE_CWD`**: 현재 작업 디렉토리를 숨기는 환경변수. 민감한 경로를 노출하고 싶지 않은 환경에서 활용
- **`--print` 모드 에이전트 프론트매터 적용**: `--print` 모드에서도 에이전트의 `tools:`와 `disallowedTools:` 프론트매터가 적용됨
- **PowerShell 자동 승인**: PowerShell 도구 명령을 권한 모드에서 자동 승인 가능
- **CRLF 붙여넣기 수정**: Windows에서 복사한 텍스트의 CRLF 처리 개선
- **슬래시 명령 제안 개선**: 슬래시 명령 자동완성과 피커 UI가 개선됨
- **서브에이전트/SDK MCP 재설정**: 서브에이전트와 SDK에서 MCP 서버 재설정 시 병렬 연결 지원

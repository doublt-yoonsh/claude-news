**핵심 기능**:

**파트너 통합**: Canva(편집 가능 프로젝트 전환), Brilliant(인터랙티브 프로토타입 변환), Datadog(1주 워크플로우를 단일 대화로 압축 데모)

### v2.1.113 — 네이티브 바이너리 전환 & 도메인 차단 설정 (4/17)

v2.1.113에서 CLI가 번들된 JavaScript 대신 **플랫폼별 네이티브 바이너리**를 생성합니다. 시작 시간과 메모리 사용량 개선이 기대됩니다.

새로운 `sandbox.network.deniedDomains` 설정으로 `allowedDomains` 와일드카드 내에서도 특정 도메인을 차단할 수 있습니다.

```json
// settings.json — 특정 도메인 차단
{
  "sandbox": {
    "network": {
      "deniedDomains": ["evil.com", "*.malicious.io"]
    }
  }
}
```

엔터프라이즈 환경에서 외부 도메인 접근을 세밀하게 제어해야 하는 경우 즉시 유용합니다.

## 개발자 워크플로우 팁

### Boris Cherny의 Opus 4.7 실전 팁 — 10~15개 세션 병렬 운영 (4/16)

Claude Code 창시자 Boris Cherny가 Opus 4.7을 수주간 독포딩(dogfooding)한 후 실전 팁을 공유했습니다.

**핵심 워크플로우**:
- **10~15개 세션 동시 운영**: 터미널 탭 5개 + 웹 5~10개, 각 세션에 별도 git worktree 할당하여 변경 충돌 방지
- **`/focus` 모드**: 중간 도구 호출·파일 읽기를 숨기고 최종 결과만 표시. "모델이 올바른 명령을 실행할 것이라 신뢰하고 결과만 확인"
- **검증 패턴 2~3배 강화**: Opus 4.7에서 검증이 2~3배 더 효과적. 백엔드는 서버 구동 후 E2E 테스트, 프론트엔드는 Claude Chromium 확장 활용
- **Auto 모드 병렬화**: 권한 프롬프트를 모델 분류기가 판단하므로 babysitting 없이 여러 Claude 동시 실행

단일 세션에서 하나씩 작업하는 패턴에서, 병렬 worktree + auto 모드로 처리량을 극대화하는 패턴으로의 전환을 보여줍니다.

[Boris Cherny Threads](https://www.threads.com/@boris_cherny/post/DXM_ATtjxna/) | [Best Practices](https://github.com/shanraisshan/claude-code-best-practice/blob/main/tips/claude-boris-6-tips-16-apr-26.md)

**Anthropic 대응**: "토큰 소비 증가를 보상하기 위해 구독자 레이트 리밋을 인상할 것"이라고 발표했습니다.

4/14의 effort 기본값 "medium" 하향 논란에 이어, Opus 4.7의 토큰 소비 증가까지 겹치면서 비용 관련 불만이 누적되고 있습니다.

### Claude 4.7 토크나이저 — 실측 비용 20~30% 증가 (4/18)

독립 벤치마크에서 Opus 4.7의 토크나이저가 콘텐츠 유형별로 1.3~1.45배 더 많은 토큰을 생성하는 것이 확인되었습니다.

| 콘텐츠 유형 | 토큰 증가 배율 |
|------------|--------------|
| 기술 문서 | 1.47x |
| TypeScript 코드 | 1.36x |
| 영어 산문 | 1.20x |
| JSON | 1.13x |
| CJK(한중일) | ~1.01x |

Anthropic의 `POST /v1/messages/count_tokens` 엔드포인트로 실측한 결과이며, 세션당 20~30% 비용 증가가 예상됩니다. **한국어·일본어·중국어 콘텐츠는 거의 영향이 없다**는 점이 주목할 만합니다. 영어/코드 위주 작업에서 할당량 소진이 빨라질 수 있으므로 effort 튜닝이나 캐시 TTL 설정(`ENABLE_PROMPT_CACHING_1H=1`)을 병행하는 것이 권장됩니다.

### v2.1.113 보안 하드닝 상세

- **macOS 시스템 경로 보호**: `/private/{etc,var,tmp,home}`이 `Bash(rm:*)` 규칙의 위험 대상에 추가
- **래퍼 명령 감지**: `env`, `sudo`, `watch`, `ionice`, `setsid`로 감싸진 명령도 deny 규칙 적용
- **find 안전성**: `Bash(find:*)` 허용 규칙이 `-exec`와 `-delete` 옵션을 자동 승인하지 않음
- **MCP 동시 호출 수정**: 동시 호출 시 타임아웃 처리가 메시지 관련 워치독 해제를 방해하던 문제 수정

## 알아두면 좋은 소소한 변경사항

- **`Shift+↑/↓` 뷰포트 스크롤**: 풀스크린 모드에서 선택 영역을 확장할 때 뷰포트가 자동 스크롤
- **`Ctrl+A`/`Ctrl+E` readline 동작**: 멀티라인 입력에서 논리적 줄의 시작/끝으로 이동
- **Windows `Ctrl+Backspace`**: 이전 단어 삭제 지원 추가
- **`/loop` Esc 취소**: 대기 중인 wakeup을 Esc로 취소 가능, wakeup이 "Claude resuming /loop wakeup"으로 표시
- **긴 URL 래핑 유지**: 응답과 Bash 출력에서 여러 줄에 걸쳐 래핑되는 URL이 클릭 가능 상태 유지
- **Bash 코멘트 표시**: 첫 줄이 코멘트만인 멀티라인 명령이 트랜스크립트에서 전체 명령 표시
- **`cd <현재 디렉토리> && git ...` 권한 제거**: 현재 디렉토리에서의 `cd && git` 패턴이 더 이상 권한 프롬프트를 발생시키지 않음
- **`OTEL_LOG_RAW_API_BODIES`**: API 요청/응답 전체 로깅을 위한 디버깅 환경변수 추가

## 주요 신규 기능 & 실전 활용

### 기본 effort 레벨 medium → high로 상향 (v2.1.94)

v2.1.94부터 API 키, Bedrock/Vertex/Foundry, Team, Enterprise 사용자의 기본 effort 레벨이 medium에서 **high**로 변경되었습니다. 이전에는 `/effort high`를 명시적으로 설정해야 더 정밀한 응답을 받을 수 있었지만, 이제 기본값으로 적용됩니다.

- **영향**: 더 상세하고 정밀한 응답이 기본값 — 복잡한 작업에서의 완성도 향상
- **토큰 증가**: high 모드는 medium 대비 더 많은 토큰을 사용하므로, 비용에 민감한 경우 `/effort medium`으로 되돌릴 수 있습니다
- **대상**: API 키 사용자, Bedrock/Vertex/Foundry, Team, Enterprise — Pro 구독 사용자는 변경 없음
- **조절**: `/effort low|medium|high` 명령으로 세션 중 언제든 변경 가능

```bash
# 비용 절약이 필요하면 medium으로 전환
/effort medium

# 복잡한 아키텍처 설계 시 high 유지 (이제 기본값)
/effort high
```

## 개발자 워크플로우 팁

### Amazon Bedrock Mantle 통합 — 환경변수 하나로 연결 (v2.1.94)

v2.1.94에서 Amazon Bedrock의 차세대 인프라인 Mantle을 통한 Claude Code 사용이 지원되었습니다. 기존 Bedrock 설정에 환경변수 하나만 추가하면 됩니다.

- **설정**: `CLAUDE_CODE_USE_MANTLE=1` 환경변수 추가
- **전제 조건**: 기존 Bedrock 인증(AWS credentials)이 설정되어 있어야 함
- **이점**: Mantle의 최적화된 추론 인프라를 통한 성능 향상

```bash
# .bashrc 또는 .zshrc에 추가
export CLAUDE_CODE_USE_MANTLE=1

# 또는 일회성 실행
CLAUDE_CODE_USE_MANTLE=1 claude
```

[GitHub v2.1.94](https://github.com/anthropics/claude-code/releases)

### CJK/멀티바이트 텍스트 손상 수정 — 한국어 사용자 필수 업데이트 (v2.1.94)

v2.1.94에서 stream-json 입출력의 CJK/멀티바이트 텍스트 손상 문제가 수정되었습니다. 한국어, 일본어, 중국어로 작업하는 개발자에게 중요한 안정성 개선입니다.

- **증상**: 한글, 한자, 일본어 등 멀티바이트 문자가 스트림 처리 중 깨지거나 잘리는 현상
- **원인**: stream-json 파서가 멀티바이트 문자의 바이트 경계를 올바르게 처리하지 못함
- **v2.1.89 수정과의 차이**: 프롬프트 히스토리에서 CJK/이모지 손실은 이미 수정되었으나, 이번에는 스트림 처리 레벨에서의 추가 수정

한국어 코드 주석이나 문서를 작성하는 프로젝트에서 작업한다면 즉시 업데이트를 권장합니다.

```bash
brew upgrade --cask claude-code
```

## 주요 신규 기능 & 실전 활용

### Skills Marketplace 생태계 본격 확장 — 423개 플러그인, 2,849개 스킬 (4/26)

Claude Code의 플러그인·스킬 생태계가 빠르게 성장하고 있습니다. 커뮤니티 기반 마켓플레이스에서 **423개 플러그인, 2,849개 스킬, 177개 에이전트**가 공개되어 있으며, Anthropic 공식 마켓플레이스 외에도 다수의 서드파티 허브가 운영 중입니다.

핵심은 **Execution Runtime** 패턴입니다. 전체 코드베이스를 컨텍스트에 로드하는 대신, Claude가 로컬에서 Python 스크립트를 실행하여 API에 접근하는 방식으로 토큰 소비를 대폭 줄입니다. 스킬은 `SKILL.md` 파일에 YAML 프론트매터와 지시사항을 정의하면 대화 컨텍스트에 따라 자동으로 트리거됩니다.

```bash
# 플러그인 검색 및 설치
claude plugin search "code-review"
claude plugin install @marketplace/code-review-pro

# 개인 스킬 생성
mkdir -p ~/.claude/skills
# → SKILL.md에 트리거 조건과 지시사항 정의
```

`blockedMarketplaces`와 `strictKnownMarketplaces` 설정으로 조직 내 승인된 마켓플레이스만 허용할 수 있습니다. "Anthropic Verified" 배지가 있는 플러그인은 수동 품질·안전 검토를 통과한 것입니다.

[BrightCoding](https://www.blog.brightcoding.dev/2026/04/26/claude-skills-marketplace-the-essential-plugin-hub-for-developers) | [GitHub](https://github.com/jeremylongshore/claude-code-plugins-plus-skills)

## 개발자 워크플로우 팁

### Context7 플러그인으로 문서 환각 제거하기

Claude가 프레임워크 API를 잘못 생성하는 가장 큰 원인은 학습 데이터의 버전 불일치입니다. **Context7 플러그인**은 라이브러리 문서를 버전별로 인덱싱하여, Claude가 필요한 페이지만 정확히 가져오도록 합니다.

예를 들어 Next.js 15, React 19, Tailwind CSS 4 등의 최신 API를 사용할 때, Context7이 현재 버전의 공식 문서를 직접 제공하므로 deprecated 패턴이나 존재하지 않는 API를 생성하는 문제가 크게 줄어듭니다.

```json
// settings.json에 Context7 MCP 서버 추가
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp"]
    }
  }
}
```

온라인 문서를 직접 검색하게 하는 것보다 Context7을 통하는 것이 효율적입니다. Claude가 직접 fetch·파싱·필터링하는 과정이 생략되어 토큰과 시간을 모두 절약합니다.

[Context7 GitHub](https://github.com/upstash/context7) | [Claude Plugin](https://claude.com/plugins/context7)

### 도구는 적을수록 좋다 — "소수 정예 도구" 원칙

MCP 서버와 플러그인이 늘어나면서 "도구 과부하" 문제가 부상하고 있습니다. Marmelab 팀의 실전 경험에 따르면, 각 도구는 컨텍스트에 복잡성을 추가하고 관리 부담을 증가시킵니다.

**실전 기준**: 실제로 주 3회 이상 사용하는 도구만 활성화하고, 나머지는 비활성화하십시오. 10개의 반쯤 쓰는 도구보다 4~5개의 잘 선택된 도구가 훨씬 효과적입니다. 프로젝트별로 `.claude/settings.local.json`에서 필요한 MCP 서버만 활성화하는 것이 권장됩니다.

[Marmelab](https://marmelab.com/blog/2026/04/24/claude-code-tips-i-wish-id-had-from-day-one.html)

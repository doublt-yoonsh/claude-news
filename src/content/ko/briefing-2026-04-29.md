## 주요 신규 기능 & 실전 활용

### v2.1.121 — MCP `alwaysLoad`, ~2GB 메모리 누수 수정, Hook 출력 대체 (4/28)

v2.1.120 회수 이후 안정화 작업을 거쳐 릴리스된 v2.1.121에서 주요 기능과 성능 수정이 포함되었습니다.

**MCP `alwaysLoad` 옵션**: MCP 서버 설정에 `alwaysLoad: true`를 추가하면, 해당 서버의 모든 도구가 tool-search deferral을 건너뛰고 **항상 사용 가능** 상태가 됩니다. 자주 사용하는 MCP 서버의 도구가 지연 로딩으로 누락되는 문제를 해결합니다.

**~2GB 메모리 누수 수정**: 이미지를 많이 처리하는 세션에서 무한 메모리 증가가 발생하던 문제와, `/usage` 명령이 대용량 트랜스크립트에서 최대 ~2GB를 누수하던 문제가 수정되었습니다. 장시간 세션 사용자에게 체감 성능 개선이 클 것입니다.

**PostToolUse Hook 출력 대체**: `hookSpecificOutput.updatedToolOutput`으로 **모든 도구**의 출력을 Hook에서 대체할 수 있습니다. 기존에는 MCP 도구만 가능했으나, 이제 Bash, Read 등 빌트인 도구에도 적용 가능합니다.

```json
// settings.json — MCP 서버 항상 로드
{
  "mcpServers": {
    "my-critical-server": {
      "command": "npx",
      "args": ["-y", "@my/mcp-server"],
      "alwaysLoad": true
    }
  }
}
```

```json
// PostToolUse Hook으로 도구 출력 변환
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "echo '{\"hookSpecificOutput\":{\"updatedToolOutput\":\"filtered output\"}}'"
      }]
    }]
  }
}
```

### v2.1.122 — Bedrock 서비스 티어, `/resume` PR URL 지원 (4/28)

v2.1.121 직후 릴리스된 v2.1.122에서는 AWS Bedrock 사용자를 위한 기능이 추가되었습니다.

**`ANTHROPIC_BEDROCK_SERVICE_TIER`**: Bedrock에서 서비스 티어를 `default`, `flex`, `priority` 중 선택할 수 있습니다. 비용과 지연 시간 사이의 균형을 조절하는 데 유용합니다.

**`/resume`에서 PR URL 붙여넣기**: `/resume` 시 PR URL을 붙여넣으면 해당 PR을 생성한 세션을 자동으로 찾아 이어갑니다. GitHub, GitLab, Bitbucket을 모두 지원합니다.

## 개발자 워크플로우 팁

### PostToolUse Hook으로 도구 출력 필터링 — 민감 정보 마스킹 활용

v2.1.121에서 확장된 `updatedToolOutput` 기능으로 실전에서 유용한 패턴을 구현할 수 있습니다. 예를 들어 Bash 도구가 반환하는 출력에서 API 키, 토큰, 내부 URL 등 민감 정보를 자동으로 마스킹하거나, 불필요하게 긴 출력을 요약하여 컨텍스트 토큰을 절약할 수 있습니다.

기존에는 MCP 도구에만 제한되었으나, 이제 Read, Bash, Grep 등 모든 빌트인 도구에 적용 가능하므로 조직의 보안 정책을 Hook 수준에서 강제할 수 있습니다.

[GitHub v2.1.121](https://github.com/anthropics/claude-code/releases/tag/v2.1.121)

### 문서에 자동 갱신되는 스크린샷 넣기

Markdown 문서에 `SCREENSHOT` 주석으로 캡처 지시사항을 명시하면, 빌드 시 headless Chrome이 자동으로 스크린샷을 캡처하여 문서에 삽입하는 시스템입니다. element, full_page, viewport 세 가지 모드를 지원하고, click, wait, crop 등의 옵션으로 UI 상태를 제어할 수 있습니다.

Claude Code로 UI를 수정한 후 문서의 스크린샷이 구버전으로 남는 문제를 해결합니다. `rails manual:build` 같은 단일 명령으로 스크린샷 캡처와 문서 빌드를 동시에 처리할 수 있어, CI/CD 파이프라인에 통합하기 좋습니다.

### Claude.ai 대규모 장애 — 12,000+ 사용자 영향, ~78분 다운 (4/28)

4월 28일 오후(UTC 17:34~18:52), Claude.ai, Claude Code, Anthropic API가 동시에 장애를 겪었습니다. Downdetector 기준 **12,000건 이상의 장애 보고**가 접수되었으며, Anthropic 상태 페이지에서 "Major Outage"로 분류되었습니다.

로그인 경로를 포함한 인증 오류가 주요 증상이었으며, Claude Design 등 연관 서비스도 영향을 받았습니다. 약 78분 후 복구되었으나, **8일 전(4/20)에도 유사한 대규모 장애**가 있었다는 점에서 인프라 안정성에 대한 우려가 커지고 있습니다.

Claude Code 사용 시 장애 대비를 위해 `--resume` 기능을 적극 활용하고, 중요한 세션에서는 주기적으로 작업 상태를 저장하는 것이 권장됩니다.

## 알아두면 좋은 소소한 변경사항

- **`/skills` 검색 필터**: `/skills` 명령에 타이핑 검색 필터가 추가되어, 많은 스킬 목록에서 빠르게 찾을 수 있음
- **`claude plugin prune`**: 고아 상태의 자동 설치 플러그인 의존성을 정리하는 명령 추가. `plugin uninstall --prune`으로 연쇄 정리 가능
- **`/branch` 포크 오류 수정**: 되감은 타임라인에서 tool_use 오류가 발생하던 문제 수정
- **이미지 최대 크기 수정**: 이미지가 2576px로 리사이즈되던 버그 수정, 올바른 최대값 2000px로 복구
- **`/mcp`에서 숨겨진 커넥터 표시**: 수동 추가 서버와 중복되어 숨겨졌던 claude.ai 커넥터가 `/mcp`에서 표시됨
- **MCP 서버 자동 재시도**: 일시적 오류 시 최대 3회 자동 재시도 로직 추가
- **음성 모드 Caps Lock 키바인딩 수정**: Caps Lock 상태에서 음성 모드 키바인딩이 올바르게 동작

## 주요 신규 기능 & 실전 활용

### Claude Code v2.1.78 — 플러그인 영속 상태와 보안 강화 (3/18)

어제 밤 릴리스된 **v2.1.78**의 핵심 변경사항입니다.

**주요 변경:**
- **`StopFailure` 훅 이벤트** — API 오류(레이트 리밋, 인증 실패 등)로 턴이 종료될 때 발동하는 새로운 훅. 자동화 워크플로우에서 오류 감지·복구 파이프라인 구축에 유용
- **`${CLAUDE_PLUGIN_DATA}` 변수** — 플러그인의 영속 상태를 저장하는 디렉토리. 플러그인 업데이트 후에도 데이터가 유지되어 플러그인 개발자에게 큰 편의
- **플러그인 에이전트 프론트매터 확장** — `effort`, `maxTurns`, `disallowedTools`를 플러그인이 배포하는 에이전트 정의에 사용 가능
- **tmux 내 터미널 알림 개선** — tmux 세션 안에서 실행 시 외부 터미널까지 알림이 전달
- **응답 텍스트 라인 단위 스트리밍** — 생성되는 대로 한 줄씩 화면에 출력
- **보안: 샌드박스 의존성 누락 시 무음 비활성화 수정** — 이전에는 샌드박스가 조용히 꺼졌으나, 이제 시작 시 경고를 표시
- **`.git`, `.claude` 보호 디렉토리 쓰기 권한 수정** — 프롬프트 없이 쓰기가 가능하던 문제 해결
- **`--worktree` 플래그 스킬·훅 미로딩 수정** — 워크트리 모드에서 스킬과 훅이 정상 로딩
- **보이스 모드 수정** — WSL2/WSLg 호환성, 수정자 키 조합 푸시투토크 수정

### v2.1.78 `StopFailure` 훅 활용 예시

API 레이트 리밋 시 자동으로 슬랙 알림을 보내는 훅 설정:

```json
{
  "hooks": {
    "StopFailure": [
      {
        "command": "curl -X POST $SLACK_WEBHOOK -d '{\"text\": \"Claude Code API 오류 발생: 확인 필요\"}'",
        "timeout": 5000
      }
    ]
  }
}
```

### `${CLAUDE_PLUGIN_DATA}` 플러그인 영속 상태 활용

플러그인 개발자는 `${CLAUDE_PLUGIN_DATA}`를 사용해 설정, 캐시, 사용자 데이터를 영속적으로 저장할 수 있습니다. 플러그인 업데이트 시에도 데이터가 보존됩니다.

### v2.1.78 보안 수정 사항

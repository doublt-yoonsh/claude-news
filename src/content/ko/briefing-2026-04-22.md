## 주요 신규 기능 & 실전 활용

### v2.1.117 — 네이티브 검색 엔진 전환 & effort 기본값 상향 (4/22)

v2.1.117에서 Glob과 Grep 도구가 내장된 **네이티브 `bfs`/`ugrep` 바이너리**로 교체되었습니다. macOS/Linux에서 파일 검색과 코드 검색 속도가 개선됩니다.

**Pro/Max 기본 effort 상향**: Opus 4.6과 Sonnet 4.6에서 기본 effort가 `medium`에서 `high`로 상향되었습니다. 4/14의 effort 기본값 하향 논란과 v2.1.111의 `xhigh` 도입에 이어, 유료 구독자의 기본 추론 품질이 다시 높아졌습니다.

**`/resume` stale 세션 요약**: 오래된 대규모 세션을 재개할 때, 전체 세션을 다시 읽기 전에 요약을 먼저 제안합니다. 컨텍스트를 효율적으로 사용하면서 이전 작업을 빠르게 파악할 수 있습니다.

**`/model` 선택 영구 저장**: 프로젝트가 특정 모델을 고정해도, 사용자가 `/model`로 선택한 모델이 재시작 후에도 유지됩니다. 시작 헤더에 모델 핀 소스가 표시됩니다.

```bash
# 모델 선택이 재시작 후에도 유지
/model  # → 선택한 모델이 다음 세션에도 적용

# stale 세션 재개 시 요약 먼저 제안
claude --resume
# → "이 세션은 3일 전 것입니다. 요약하시겠습니까?"
```

### Two Strikes Rule — 두 번째 실패 후 즉시 리셋

Claude가 동일한 이슈를 두 번 연속 실패하면, 같은 대화에서 계속 시도하지 말고 `/clear`로 리셋한 후 개선된 프롬프트로 재시작해야 합니다.

**이유**: 실패한 접근이 컨텍스트에 누적되면 "컨텍스트 오염"이 발생하여, Claude가 같은 실수를 반복하는 경향이 강화됩니다. 세 번째 시도보다 깨끗한 재시작이 훨씬 효과적입니다.

```bash
# 두 번 실패 후
/clear
# → 문제를 재구성하여 새로운 접근으로 시작
```

## 보안/제한 이슈

### MCP 프로토콜 설계 취약점 — 150M+ 다운로드에 영향 (4/20)

OX Security가 Anthropic의 MCP(Model Context Protocol) 설계 자체에 존재하는 **구조적 취약점**을 공개했습니다. 코딩 버그가 아닌 아키텍처 결함입니다.

**기술 상세**:
- STDIO 전송 방식이 설정에서 직접 명령 실행으로 연결되는 구조
- 공격자가 MCP 설정에 영향을 줄 수 있으면 호스트에서 **임의 셸 명령 실행(RCE)** 가능
- 모든 공식 SDK(Python, TypeScript, Java, Rust)에 영향
- 7,000개 이상의 공개 MCP 서버, 150M+ 패키지 다운로드, 최대 200,000개 취약 인스턴스

**영향 범위**: Claude Code, Cursor, VS Code Claude 확장, Windsurf, Gemini CLI, LiteLLM, LangChain 등 MCP를 구현한 모든 도구.

**Anthropic 대응**: 프로토콜 수정을 거부하고, 입력 검증은 개발자의 책임이라고 답변했습니다.

신뢰할 수 없는 출처의 MCP 서버 설정을 사용하지 않도록 주의해야 합니다. `sandbox.network.deniedDomains`(v2.1.113)로 도메인을 제한하는 것이 권장됩니다.

[The Hacker News](https://thehackernews.com/2026/04/anthropic-mcp-design-vulnerability.html) | [OX Security](https://www.ox.security/blog/the-mother-of-all-ai-supply-chains-critical-systemic-vulnerability-at-the-core-of-the-mcp/)

## 알아두면 좋은 소소한 변경사항

- **`/model` 선택 영구 저장**: 프로젝트 모델 핀과 별개로 사용자 선택이 재시작 후에도 유지. 시작 헤더에 모델 핀 소스 표시
- **`cleanupPeriodDays` 확장**: `~/.claude/tasks/`, `shell-snapshots/`, `backups/`도 자동 정리 대상에 포함
- **Advisor Tool "experimental" 라벨**: 실험적 기능임을 명시하는 라벨 추가, "content could not be processed" 오류 수정
- **Opus 4.7 컨텍스트 윈도우 수정**: 200K로 잘못 계산되던 문제를 1M으로 수정
- **OAuth 토큰 자동 갱신**: 401 응답 시 자동으로 토큰 갱신
- **WebFetch 행 수정**: 대규모 페이지에서 WebFetch가 멈추던 문제 해결
- **`NO_PROXY` 준수**: Bun 환경에서 `NO_PROXY` 환경변수가 정상 동작
- **플러그인 마켓플레이스**: `blockedMarketplaces`/`strictKnownMarketplaces` 설정 강제 적용

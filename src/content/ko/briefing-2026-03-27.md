## 주요 신규 기능 & 실전 활용

### Windows PowerShell 도구 — opt-in 프리뷰 (v2.1.84)

Windows 사용자가 기다리던 네이티브 PowerShell 지원이 opt-in 프리뷰로 도입되었습니다. 기존 WSL이나 Git Bash에 의존하지 않고 PowerShell을 직접 사용할 수 있습니다.

- Windows 환경에서 네이티브 명령어 실행 가능
- opt-in 방식이므로 설정에서 활성화 필요
- Bash 도구와 동일한 권한 모델 적용

### 조건부 훅 — `if` 필드 도입 (v2.1.85)

훅에 `if` 필드가 추가되어 permission rule 문법으로 조건부 실행이 가능합니다. 모든 도구 호출마다 훅 프로세스를 스포닝하던 오버헤드가 크게 줄어듭니다.

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "if": "tool == 'Bash'",
        "command": "echo 'Bash 명령 감지' >&2"
      }
    ]
  }
}
```

이전에는 모든 도구 호출에서 훅이 실행되었지만, 이제 조건을 지정하여 필요한 경우에만 트리거됩니다.

### MCP 멀티서버 환경변수 (v2.1.85)

`CLAUDE_CODE_MCP_SERVER_NAME`과 `CLAUDE_CODE_MCP_SERVER_URL` 환경변수가 추가되어 여러 MCP 서버를 동적으로 구성할 수 있습니다. CI/CD나 멀티 환경에서 MCP 서버를 코드 변경 없이 전환할 수 있습니다.

```bash
export CLAUDE_CODE_MCP_SERVER_NAME="staging-server"
export CLAUDE_CODE_MCP_SERVER_URL="http://localhost:3000"
```

## 개발자 워크플로우 팁

### Anthropic의 장시간 에이전트 하네스 설계 패턴

Anthropic 엔지니어링 블로그에서 장시간 자율 코딩 세션을 위한 **3-에이전트 아키텍처**를 공개했습니다.

- **Planner**: 작업 분해 및 실행 계획 수립
- **Generator**: 실제 코드 생성 및 수정
- **Evaluator**: 생성된 결과의 품질 평가 및 피드백

GAN에서 영감을 받은 이 패턴은 프론트엔드 디자인 품질과 자기 평가 정확도를 동시에 높입니다. 멀티시간 코딩 세션에서 품질 저하 없이 자율 작업을 유지하는 핵심 설계 기법입니다.

[Anthropic Engineering Blog](https://www.anthropic.com/engineering/harness-design-long-running-apps) | [GeekNews](https://news.hada.io/topic?id=27457)

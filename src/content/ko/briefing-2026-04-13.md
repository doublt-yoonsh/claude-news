## 주요 신규 기능 & 실전 활용

### Claude Code `/advisor` 슬래시 명령 — v2.1.101에서 사용 가능

v2.1.101에 추가된 `/advisor` 명령이 주목받고 있습니다. Anthropic의 Advisor Strategy(4/9 발표)를 Claude Code에서 직접 사용할 수 있는 기능입니다.

**동작 방식**:
1. Claude Code에서 `/advisor`를 실행하고 모델 선택 (Sonnet 4.6 또는 Opus 4.6)
2. 이후 Claude Code가 복잡한 결정, 반복 오류, 작업 완료 체크포인트에서 자동으로 어드바이저 모델을 호출
3. 어드바이저는 전체 대화 트랜스크립트(원본 요청, 도구 호출, 파일 읽기, 명령 출력, 추론 과정)에 접근
4. 단, 어드바이저는 파일 시스템, 셸, 웹, MCP 서버에 직접 접근 불가

**권장 구성**: Sonnet 4.6을 메인 모델로 사용하고 Opus 4.6을 어드바이저로 설정하면, 비용 효율성과 핵심 결정의 추론 깊이를 동시에 확보할 수 있습니다.

**성능 수치**: SWE-bench Multilingual에서 Sonnet + Opus 어드바이저가 Sonnet 단독 대비 2.7 포인트 향상 (74.8% vs 72.1%), 에이전트 작업당 비용은 11.9% 절감.

[Claude Blog](https://claude.com/blog/the-advisor-strategy) | [Advisor Tool Docs](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool)

## 개발자 워크플로우 팁

### `/advisor`를 활용한 비용 최적화 워크플로우

Claude Code에서 Opus 4.6을 기본 모델로 사용하는 팀이라면, `/advisor`를 통해 역전 구성을 시도해볼 만합니다:

```bash
# Claude Code에서 /advisor 실행 후 Opus 4.6 선택
# → 기본 모델은 Sonnet 4.6, 어드바이저는 Opus 4.6
# → 일상적 코딩은 Sonnet이 처리, 핵심 아키텍처 결정 시 Opus가 자문

# 효과:
# - SWE-bench 기준 성능 하락 2.7%p에 불과
# - 비용 11.9% 절감
# - Haiku + Opus 어드바이저 조합은 비용 85% 절감 (단, 성능 하락 큼)
```

특히 대규모 코드베이스에서 반복적 리팩토링을 수행할 때, Sonnet이 기본 실행을 담당하고 Opus가 "이 리팩토링 방향이 맞는가?"를 판단하는 패턴이 비용 대비 효과적입니다.

[Claude Blog](https://claude.com/blog/the-advisor-strategy)

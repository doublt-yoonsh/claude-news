## 개발자 워크플로우 팁

### CLAUDE.md 최적화 — 200줄 이내, ~100개 지시사항 예산

Claude Code 공식 Best Practices 가이드에서 CLAUDE.md 작성의 정량적 기준이 정리되었습니다.

**핵심 수치**:
- **시스템 프롬프트가 ~50개 지시사항을 소비** → 프로젝트에 할당 가능한 지시사항은 **~100개**
- **200줄 이내** 유지 — 길어질수록 지시사항 준수율 하락
- **컨텍스트 90% 이상** 사용 시 출력 품질 저하

**실전 패턴**:
- 코드 복사 대신 `src/auth.ts:42` 형태의 **file:line 포인터**로 참조
- 도메인별 지식은 CLAUDE.md가 아닌 **스킬로 분리**하여 필요 시에만 로드
- 빌드·테스트 명령, 핵심 컨벤션을 최상단에 간결하게 배치

```markdown
# 좋은 예 — 간결한 CLAUDE.md
## 빌드
- `pnpm dev` — 개발 서버
- `pnpm test` — 테스트

## 컨벤션
- 새 API: src/routes/api.ts:15 패턴 참조
- 에러 처리: src/lib/errors.ts 참조
```

"지시사항을 추가할 때마다 다른 지시사항의 준수율이 미세하게 감소한다"는 점을 기억해야 합니다. CLAUDE.md가 비대해지면 실질적으로 사용 가능한 컨텍스트가 줄어드는 역효과가 발생합니다.

### Opus 4.6 vs 4.7 토큰 비용 계산기 — 프롬프트별 실제 비용 차이 확인

GeekNews에 소개된 토큰 비용 계산기로, 자신의 프롬프트가 Opus 4.7에서 얼마나 더 많은 토큰을 소비하는지 즉시 확인할 수 있습니다.

**실측 결과**:
- 평균 **37.4%** 토큰 증가
- 범위: +19.0% ~ +86.2% (프롬프트 유형별 편차)

어제 보도된 "20~30% 비용 증가" 예측보다 실제 차이가 더 클 수 있음을 보여줍니다. 자신의 주요 프롬프트를 입력하여 실제 영향을 파악하고, `ENABLE_PROMPT_CACHING_1H=1`이나 effort 조절로 비용을 관리하는 근거 자료로 활용할 수 있습니다.

### Claude Sonnet 4.5/4 1M 컨텍스트 베타 — 4월 30일 종료 예정

Claude Sonnet 4.5와 Sonnet 4의 1M 컨텍스트 윈도우 베타가 4월 30일에 종료됩니다. 이후 해당 모델의 1M 컨텍스트 요청은 지원되지 않습니다.

4/17 발표된 Sonnet 4/Opus 4의 6월 15일 API 지원 종료와 별도의 일정입니다. 1M 컨텍스트가 필요한 경우 Opus 4.6 또는 4.7로 마이그레이션이 권장됩니다.

### Claude Sonnet 3.7 / Haiku 3.5 공식 퇴역

Claude Sonnet 3.7과 Haiku 3.5가 공식 퇴역했습니다. 해당 모델에 대한 API 요청은 에러를 반환합니다. Sonnet 3.7은 Sonnet 4.6 이상으로, Haiku 3.5는 Haiku 4.5로 즉시 마이그레이션이 필요합니다. Claude Code에서 `ANTHROPIC_MODEL`로 이전 모델을 지정한 경우 설정 업데이트가 필요합니다.

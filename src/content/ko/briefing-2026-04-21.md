## 주요 신규 기능 & 실전 활용

### v2.1.116 — 대규모 세션 복원 67% 가속 & MCP 지연 로딩 (4/20)

v2.1.116에서 40MB 이상 대규모 세션의 `/resume` 속도가 최대 67% 개선되었습니다. 장기 프로젝트나 다중 에이전트 세션에서 세션 재개가 눈에 띄게 빨라집니다.

**MCP 시작 최적화**: MCP 서버 시작 시 `resources/templates/list` 호출을 지연하여 초기 로딩 시간이 단축되었습니다. 다수의 MCP 서버를 연결한 환경에서 체감 차이가 큽니다.

**thinking 스피너 인라인 진행 표시**: 대기 중 스피너에 "still thinking", "thinking more", "almost done thinking" 등 상황별 힌트가 표시됩니다.

**보안 개선**: 샌드박스 자동 허용이 `/`, `$HOME`, 주요 시스템 디렉토리를 대상으로 하는 `rm`/`rmdir`에 대해 위험 경로 안전장치를 우회하지 않도록 수정되었습니다.

```bash
# 대규모 세션 빠르게 재개
claude --resume

# /config에서 옵션 값 검색 가능
/config  # 이제 옵션 이름뿐 아니라 값도 매칭
```

## 개발자 워크플로우 팁

### Claude Code vs Codex — 14년차 시니어 엔지니어의 100시간+ 실전 비교

GeekNews에서 55포인트를 기록한 비교 분석입니다. 80,000줄 Python/TypeScript 프로젝트에서 Claude Code(Opus 4.6)와 Codex(GPT-5.4)를 각각 100시간+, 20시간+ 사용한 결과입니다.

**핵심 비교**:
- **Claude Code**: 빠른 프로토타이핑에 탁월하지만 적극적 관리 필요. 지시사항 무시, 작업 미완료, 실패 테스트 무단 수정 경향
- **Codex**: 3~4배 느리지만 체계적. 아키텍처 준수, 자기 수정, AGENTS.md에서 절대 이탈 않음
- **하이브리드 전략**: Claude로 초안 → Codex로 리뷰가 최적

**실전 결론**: 바이브 코딩과 빠른 이터레이션에는 Claude Code, 엔터프라이즈 품질과 아키텍처 일관성이 중요한 시스템에는 Codex. 두 도구의 장단점을 파악하고 상황에 맞게 전환하는 것이 핵심입니다.

[GeekNews](https://news.hada.io/topic?id=28538)

### Planning-First 접근법 — 안내 없는 시도는 33% 성공률

Anthropic 내부 테스트에서 사전 계획 없이 Claude에게 바로 구현을 맡기면 성공률이 약 33%에 그치는 것으로 나타났습니다.

**실전 적용**:
1. "don't implement yet" — 구현 전 계획 모드 강제
2. 리서치/분석 단계와 구현 단계를 분리
3. CLAUDE.md에 아키텍처 포인터를 간결하게 유지하여 모델이 컨텍스트를 빠르게 파악하도록 구성

```bash
# plan 모드로 먼저 분석
claude "이 코드베이스를 분석하고 리팩토링 계획을 세워줘. 아직 구현하지 마."
```

## 알아두면 좋은 소소한 변경사항

- **`/config` 값 검색**: `/config`에서 옵션 이름뿐 아니라 옵션 값으로도 검색 가능
- **`/doctor` 응답 중 실행 가능**: Claude가 응답 중일 때도 `/doctor`를 열 수 있음
- **`/reload-plugins` 자동 설치**: 누락된 플러그인 의존성을 자동으로 설치
- **GitHub API 레이트 리밋 힌트**: Bash 도구에서 GitHub API 레이트 리밋 발생 시 힌트 표시
- **에이전트 프론트매터 `hooks:`**: `--agent`로 메인 스레드 에이전트로 실행할 때도 hooks가 동작
- **데바나가리/인도 문자 렌더링 수정**: 인도 계열 문자가 정상 렌더링
- **Kitty 프로토콜 터미널**: `Cmd+Left/Right` 동작 수정

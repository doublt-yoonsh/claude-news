## 주요 신규 기능 & 실전 활용

### Prompt Caching TTL 설정 — 토큰 비용 직접 제어 (v2.1.108)

v2.1.108에서 프롬프트 캐싱의 TTL(Time-To-Live)을 환경변수로 직접 제어할 수 있게 되었습니다. 최근 캐시 TTL이 1시간에서 5분으로 단축되면서 Pro Max 5x 사용자들의 할당량 소진 문제가 악화되었는데, 이를 사용자가 직접 조정할 수 있습니다.

```bash
# 1시간 캐시 TTL 활성화 (구독 사용자 권장)
export ENABLE_PROMPT_CACHING_1H=1

# 5분 캐시 TTL 강제 적용 (API 키 사용자 세밀 제어)
export FORCE_PROMPT_CACHING_5M=1
```

API 키, Bedrock, Vertex, Foundry 모든 프로바이더에서 동작합니다. `DISABLE_TELEMETRY` 설정 시 1시간 캐시 대신 5분 캐시로 폴백되던 버그도 동시에 수정되었습니다. 할당량이 빠르게 소진된다면 `ENABLE_PROMPT_CACHING_1H=1`을 즉시 적용하는 것이 권장됩니다.

[GitHub v2.1.108](https://github.com/anthropics/claude-code/releases)

### /recap — 세션 복귀 시 컨텍스트 자동 요약 (v2.1.108)

장시간 자리를 비운 뒤 세션에 돌아왔을 때, `/recap` 명령으로 현재 세션의 작업 진행 상황을 요약받을 수 있습니다. Claude가 대화 히스토리를 분석하여 "무엇을 하고 있었는지, 어디까지 진행했는지, 다음에 무엇을 해야 하는지"를 정리합니다.

```bash
# 세션 복귀 시 컨텍스트 요약
/recap

# 자리 비움 감지 시 자동 요약 제공
# /config에서 설정하거나:
export CLAUDE_CODE_ENABLE_AWAY_SUMMARY=1
```

`/resume`이 세션을 이어서 여는 것이라면, `/recap`은 세션의 현재 상태를 한눈에 파악하는 기능입니다. 장기 프로젝트에서 다음 날 작업을 재개할 때 즉시 컨텍스트를 회복할 수 있습니다.

[GitHub v2.1.108](https://github.com/anthropics/claude-code/releases)

### Routines — 클라우드 기반 스케줄 자동화 (4/14)

Claude Code에 **Routines** 기능이 추가되었습니다. 반복적인 작업을 스케줄링하여 Anthropic의 웹 인프라에서 실행할 수 있으며, 로컬 머신이 꺼져 있어도 동작합니다.

기존의 `/loop`이 로컬 세션에서 동작하는 반면, Routines는 클라우드 기반으로 "매일 오전 9시에 코드 리뷰 실행", "매주 금요일에 의존성 업데이트 확인" 같은 워크플로우를 자동화합니다. Managed Agents의 Session 아키텍처를 활용한 것으로, 로컬 Claude Code와 클라우드 에이전트의 경계가 더욱 흐려지고 있습니다.

## 개발자 워크플로우 팁

### /recap + /resume 조합으로 멀티 프로젝트 세션 관리

v2.1.108의 `/recap`과 개선된 `/resume` 피커를 조합하면 멀티 프로젝트 작업에서 세션 전환이 매끄러워집니다.

```bash
# 1. 다른 프로젝트 세션으로 전환
/resume
# → 기본: 현재 디렉토리의 세션만 표시
# → Ctrl+A: 전체 프로젝트 세션 보기

# 2. 세션 전환 후 컨텍스트 파악
/recap
# → "어디까지 했는지" 즉시 확인
```

`/resume` 피커가 현재 디렉토리의 세션을 기본으로 보여주도록 변경되어, 프로젝트별 세션 관리가 한결 편해졌습니다.

### 슬래시 명령 디스커버리 — Claude가 직접 도구 호출 (v2.1.108)

v2.1.108부터 Claude가 Skill 도구를 통해 `/init`, `/review`, `/security-review` 같은 내장 슬래시 명령을 직접 호출할 수 있습니다. 이전에는 사용자가 명시적으로 입력해야 했지만, 이제 Claude가 작업 맥락에서 적절한 명령을 판단하고 자동 실행합니다.

"이 프로젝트를 초기화해줘" → Claude가 `/init` 호출. "보안 검토를 해줘" → `/security-review` 호출. 커스텀 스킬도 같은 방식으로 디스커버리됩니다.

### Plan-first 워크플로우의 정량적 근거

커뮤니티 테스트에서 AI 코딩 에이전트에게 구현을 바로 시키면 성공률이 약 33%에 불과하지만, 먼저 계획을 작성하게 하고 → 검토 → 수정 → 구현 순서를 따르면 성공률이 크게 향상된다는 결과가 공유되었습니다. `/plan` 모드나 Plan Mode를 적극 활용해야 하는 정량적 근거입니다.

### v2.1.108 주요 버그 수정 16건

안정성에 직접 영향을 주는 수정사항입니다:

- **`/login` 붙여넣기 불가 수정**: v2.1.105 회귀 — 로그인 시 인증 코드를 붙여넣을 수 없었던 문제
- **`DISABLE_TELEMETRY` 캐시 폴백**: 텔레메트리 비활성화 시 1시간 캐시 대신 5분 캐시로 폴백되던 문제
- **Auto 모드 권한 오류**: 안전 분류기 트랜스크립트가 컨텍스트 윈도우를 초과할 때 Agent 도구가 불필요한 권한 요청
- **`CLAUDE_ENV_FILE` `#` 코멘트**: 환경 파일이 `#`으로 끝나면 Bash 출력이 사라지던 문제
- **`--resume` 세션 이름 유실**: `/rename`이나 `--name`으로 지정한 이름이 복구 시 사라지던 문제
- **`--teleport` 터미널 깨짐**: 텔레포트 후 이스케이프 코드가 깨진 문자로 표시되던 문제
- **디스크 풀 시 트랜스크립트 실패 무시**: 디스크 공간 부족 시 트랜스크립트 쓰기 실패가 사일런트 드롭되던 문제

## 알아두면 좋은 소소한 변경사항

- **`/undo` = `/rewind` 별칭 추가**: 직관적인 이름으로 되돌리기 실행 가능
- **`/resume` 피커 개선**: 현재 디렉토리 세션 기본 표시, `Ctrl+A`로 전체 프로젝트 보기
- **`/model` 전환 경고**: 대화 중 모델 전환 시 다음 응답이 전체 히스토리를 캐시 없이 재읽기한다는 경고 표시
- **메모리 사용량 감소**: 언어 문법 지연 로딩으로 파일 읽기·편집·구문 강조 시 메모리 풋프린트 축소
- **에러 메시지 개선**: 서버 레이트 리밋과 플랜 사용량 제한 구분 표시, 5xx/529 에러 시 status.claude.com 링크
- **슬래시 명령 자동 제안**: 존재하지 않는 명령 입력 시 가장 유사한 명령을 제안
- **`Ctrl+O` verbose 표시기**: 상세 트랜스크립트 뷰에 "verbose" 인디케이터 추가

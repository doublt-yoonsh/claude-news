## 주요 신규 기능 & 실전 활용

### Anthropic 공식 플러그인 디렉토리 정식 운영 중

Anthropic이 관리하는 공식 플러그인 디렉토리(`claude-plugins-official`)가 GitHub에서 운영되고 있습니다. Claude Code 시작 시 자동으로 연결되며, `/plugin` 명령의 Discover 탭에서 바로 탐색할 수 있습니다.

- **구조**: `/plugins/`(Anthropic 내부 개발) + `/external_plugins/`(파트너 및 커뮤니티)
- **설치**: `/plugin install {플러그인명}@claude-plugins-official`
- **웹 카탈로그**: [claude.com/plugins](https://claude.com/plugins)에서 전체 목록 확인 가능
- 커뮤니티 마켓플레이스(claudemarketplaces.com, buildwithclaude.com 등)도 별도 운영 중

```bash
# 공식 마켓플레이스에서 플러그인 설치 예시
claude /plugin install claude-md-management@claude-plugins-official
```

[GitHub — claude-plugins-official](https://github.com/anthropics/claude-plugins-official) | [Claude Code 플러그인 문서](https://code.claude.com/docs/en/discover-plugins)

## 개발자 워크플로우 팁

### "50 Claude Code Tips and Best Practices" — 실전 팁 총정리

Builder.io가 Claude Code 일상 사용을 위한 50가지 팁을 정리했습니다. 단순 기능 소개가 아닌, 실무에서 바로 적용할 수 있는 구체적 레시피 위주입니다.

**핵심 원칙 3가지:**
1. **Plan Mode 먼저**: 멀티파일 변경, 아키텍처 결정, 익숙하지 않은 코드는 반드시 Plan Mode(Shift+Tab)로 시작
2. **CLAUDE.md는 팀의 공유 규칙**: Claude가 실수할 때마다 규칙을 추가하여 반복 방지 — git에 커밋하여 팀 전체가 공유
3. **훅은 CLAUDE.md보다 강하다**: CLAUDE.md는 권고 사항이지만, 훅은 결정적(deterministic) — 매번 반드시 실행되어야 하는 작업에 사용

**실전 레시피:**
- MCP 서버는 Playwright(브라우저 테스트), PostgreSQL(스키마 쿼리), Slack(버그 리포트), Figma(디자인→코드)부터 시작
- 10~15개 병렬 세션 + 각각 독립 git worktree로 충돌 방지
- `/compact`를 주기적으로 실행하여 토큰 소비 절약

[Builder.io — 50 Tips](https://www.builder.io/blog/claude-code-tips-best-practices) | [Builder.io — Every March Update](https://www.builder.io/blog/claude-code-updates)

## 알아두면 좋은 소소한 변경사항

- **Bedrock/Vertex/Foundry 캐시 히트율 개선**: 도구 설명에서 동적 콘텐츠를 제거하여 프롬프트 캐시 히트율이 향상되었습니다 (v2.1.86).

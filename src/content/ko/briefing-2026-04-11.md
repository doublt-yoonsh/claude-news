## 주요 신규 기능 & 실전 활용

### `/team-onboarding` — 로컬 사용 기록 기반 팀원 온보딩 가이드 생성 (v2.1.101)

v2.1.101에 추가된 `/team-onboarding` 명령은 현재 머신의 Claude Code 사용 기록을 분석하여 신규 팀원을 위한 램프업 가이드를 자동 생성합니다. 개인화된 슬래시 명령, 자주 쓰는 스킬, 프로젝트별 CLAUDE.md 패턴을 추출하여 마크다운 문서로 출력합니다.

```bash
# 기존 팀의 Claude Code 사용 패턴을 요약한 가이드 생성
/team-onboarding

# 결과를 저장소에 커밋하여 신규 팀원에게 공유
git add docs/claude-onboarding.md
```

팀 내부에 "베테랑만 아는" 암묵지를 명시화하는 용도로, 기존 팀의 워크플로우 패턴을 신규 입사자에게 전달하는 부담을 줄일 수 있습니다.

[GitHub v2.1.101](https://github.com/anthropics/claude-code/releases/tag/v2.1.101)

### OS CA 인증서 스토어 기본 신뢰 — 엔터프라이즈 TLS 프록시 즉시 지원 (v2.1.101)

Claude Code가 운영체제의 CA 인증서 스토어를 기본적으로 신뢰하도록 변경되었습니다. 자체 CA로 발급한 인증서를 사용하는 엔터프라이즈 TLS 프록시(Zscaler, Netskope, Palo Alto 등)에서 별도 설정 없이 동작합니다.

```bash
# 기본 동작 — OS CA 스토어 자동 신뢰
claude

# 번들된 CA만 사용하려면 (과거 동작으로 복귀)
export CLAUDE_CODE_CERT_STORE=bundled
claude
```

이전에는 `NODE_EXTRA_CA_CERTS` 또는 `SSL_CERT_FILE` 같은 환경변수를 수동으로 설정해야 했습니다. 보안 정책이 엄격한 사내 네트워크에서 Claude Code 온보딩 마찰이 크게 줄어듭니다.

[GitHub v2.1.101](https://github.com/anthropics/claude-code/releases/tag/v2.1.101)

### Advisor Strategy 공식 출시 — Opus 조언자 + Sonnet/Haiku 실행자 (4/10)

Anthropic이 Claude Platform 블로그에서 **Advisor Strategy**를 공식 발표했습니다. Opus를 조언자(advisor)로, Sonnet 또는 Haiku를 실행자(executor)로 페어링하여 프론티어 수준 추론을 필요할 때만 호출하는 패턴입니다.

- **Sonnet + Opus advisor**: SWE-bench Multilingual에서 **2.7%p 향상**, 비용은 **11.9% 절감**
- **Haiku + Opus advisor**: BrowseComp에서 **41.2%** (Haiku 단독 19.7% 대비 2배), Sonnet 단독 대비 **85% 비용 절감**
- **구현 방식**: Messages API 요청에 `advisor_20260301`을 선언하면 단일 API 호출 내에서 모델 핸드오프가 이루어집니다
- **사용 사례**: 실행자가 복잡한 결정을 마주할 때만 조언자에게 가이드 요청. 조언자는 도구를 직접 호출하거나 사용자 대면 출력을 생성하지 않음
- **토큰 추적**: 모델 티어별로 분리되어 비용 가시성 확보

"Opus는 모든 호출에 사용하기엔 비싸지만, 결정적 순간에 한 번씩 호출하는 방식은 매우 효율적"이라는 패턴이 공식 기능으로 정리되었습니다.

### `ant` CLI — Anthropic 공식 Claude API CLI 출시

Anthropic이 Claude Developer Platform의 공식 CLI인 `ant`을 출시했습니다. 터미널에서 Claude API를 직접 호출하고, YAML 파이프 또는 타입 플래그로 요청 본문을 구성할 수 있습니다.

```bash
# Go 1.22 이상 필요
go install github.com/anthropics/anthropic-cli/cmd/ant@latest

# Messages API 호출 예시
export ANTHROPIC_API_KEY=sk-ant-...
ant messages create \
  --model claude-opus-4-6 \
  --max-tokens 1024 \
  --message '{role: user, content: "Hello, Claude"}'

# --transform으로 응답 필드 추출
ant messages create ... --transform '.content[0].text'
```

**Claude Code 네이티브 통합**: Claude Code가 `ant`의 구조화된 출력을 자동으로 파싱·추론합니다. 커스텀 통합 코드 없이 "이 프롬프트를 Opus에게 보내서 결과를 Sonnet이 이어 처리"와 같은 파이프라인을 셸 레벨에서 구성할 수 있습니다.

[GitHub anthropics/anthropic-cli](https://github.com/anthropics/anthropic-cli) | [Claude API 문서](https://platform.claude.com/docs/en/api/sdks/cli)

### `API_TIMEOUT_MS`로 로컬 LLM·느린 게이트웨이 지원 (v2.1.101)

v2.1.101 이전에는 하드코딩된 5분 요청 타임아웃이 느린 백엔드(로컬 LLM, 확장 사고 모드, 느린 게이트웨이)를 중단시켰습니다. 이제 `API_TIMEOUT_MS` 환경변수를 존중하여 사용자가 직접 타임아웃을 조정할 수 있습니다.

```bash
# 로컬 LLM 또는 느린 엔터프라이즈 프록시에서 10분 타임아웃
export API_TIMEOUT_MS=600000
claude

# 확장 사고(extended thinking) 모드의 장시간 작업
export API_TIMEOUT_MS=1200000  # 20분
```

특히 Ollama, LM Studio, vLLM 같은 로컬 모델을 `ANTHROPIC_BASE_URL`로 연결해 쓰는 환경, 그리고 사내 게이트웨이를 거쳐 장시간 작업을 돌리는 조직에 즉시 도움이 됩니다.

### Git 히스토리를 5개 명령으로 분석하고 Claude Code에 전달하기

GeekNews에서 72포인트를 받은 "Git Commands Before Reading Code"(piechowski.io)는 낯선 코드베이스를 열었을 때 코드를 읽기 **전에** Git 히스토리로 프로젝트 건강 상태를 진단하는 5가지 명령을 제시합니다. Claude Code와 결합하면 CLAUDE.md 초기 작성이나 신규 프로젝트 인수 시점에 강력한 프리플라이트가 됩니다.

```bash
# 1. 가장 많이 변경된 파일 — 문제 핫스팟
git log --format=format: --name-only | grep -v '^$' | sort | uniq -c | sort -rg | head -20

# 2. 주요 기여자 — bus factor 확인
git shortlog -sn --all | head -10

# 3. 버그 집중 영역 — 'fix'/'bug' 커밋과 교차
git log --all --oneline --grep='fix\|bug' | wc -l

# 4. 개발 모멘텀 — 월별 커밋 수 추이
git log --format='%ad' --date=format:'%Y-%m' | sort | uniq -c

# 5. 긴급 수정 빈도 — 배포 안정성 지표
git log --all --oneline --grep='revert\|hotfix' | wc -l
```

이 결과를 `/init` 초기 컨텍스트에 포함시키면, Claude Code가 "어느 파일이 위험한지", "누구의 판단을 참조해야 하는지"를 미리 알게 됩니다. 단순 기술 지표가 아닌 **팀 데이터**를 먼저 파악한다는 관점이 핵심입니다.

[GeekNews](https://news.hada.io/topic?id=28324) | [piechowski.io](https://piechowski.io/post/git-commands-before-reading-code/)

### POSIX `which` 폴백에서 command injection 취약점 수정 (v2.1.101)

v2.1.101에서 LSP 바이너리 탐지 시 사용되는 POSIX `which` 폴백 경로에서 command injection 취약점이 수정되었습니다. 특정 조건에서 악성 PATH 엔트리가 임의 셸 명령 실행으로 이어질 수 있었습니다.

- **영향**: LSP 관련 서브시스템이 호출되는 모든 환경 (사실상 거의 모든 사용자)
- **권장 조치**: 즉시 업데이트 — `brew upgrade --cask claude-code`
- **관련 수정**: 동시에 `permissions.deny` 규칙이 PreToolUse 훅의 `permissionDecision: "ask"`를 덮어쓰지 못하던 문제도 수정되었습니다

- **`settings.json` 회복탄력성**: 알 수 없는 훅 이벤트 이름이 있어도 전체 설정 파일이 무시되지 않도록 수정
- **Bedrock/Vertex/3rd party에서 커스텀 키바인딩 로딩 수정**: `~/.claude/keybindings.json`이 비-Anthropic 프로바이더에서 로딩되지 않던 문제 수정
- **`claude -p --resume <이름>`**: `/rename`으로 지정한 세션 제목과 `--name` 플래그로 지정한 세션 이름 모두 지원
- **Bedrock SigV4 인증 수정**: `ANTHROPIC_AUTH_TOKEN`·`apiKeyHelper`·`ANTHROPIC_CUSTOM_HEADERS` 설정 시 403 오류 발생하던 문제 수정
- **리소스 정리**: SDK `query()`가 `for await` 중단·`await using` 시 서브프로세스와 임시 파일을 정리
